MultiLogic — справка по строкам логики

(файл Custom\Robots\MultiLogic_LogicHelp.html — автоматически из MultiLogic.cs;

обновляется при запуске робота и по кнопке Help; ручные правки перезаписываются)

ОБЗОР: ресурсы ПК, портфели логик, сохранение состояния

MultiLogic считает эффективный портфель отдельно для каждой логики L1…L10

(старт 0, может быть отрицательным): realized + unrealized по сделкам MultiLogic_Ln.

Файлы портфелей (лайв, не тестер):

Engine\{имя робота}_LogicPortfolio_L1.txt … _L10.txt

История до 5000 точек на логику; сброс на диск ~раз в 30 с при изменениях.

HTML-отчёт (вкладка «Отчёт», Engine\{имя}_Report.html):

Пишется в тестере, лайве и при фейке (эмулятор OsEngine, EmulatorIsOn).

В шапке — текущий режим (тестер / лайв / фейк / оптимизатор) и история смены режимов.

Графики equity, открытых/закрытых позиций от времени и opens/closes по дням;

блок «Металогика»: текущий статус, порядок приоритета L1…L10 по PnlSMA, веса и журнал входов;

таблицы по инструментам, логикам L1…L10, закрытым позициям (с длительностью), причинам закрытия.

Интервал перезаписи — параметр «HTML-отчёт: интервал (сек)» (по умолчанию 90; не на каждой сделке).

Кнопка «Открыть HTML-отчёт» (вкладка «Отчёт») — открыть файл в браузере.

На первой вкладке — «Открыть HTML-файл результатов тестирования» (то же действие).

Первая вкладка параметров:

«Сохранить настройки и результаты» — JSON (параметры, все 10 строк логик,

  • портфели, открытые позиции); файл выбираете сами.

«Загрузить настройки и результаты» — подставить всё из JSON (биржевые заявки

  • по позициям не выставляются).

Справочный % годовых (внизу вкладки): «Заполнить начальную сумму и дату портфеля»

  • — текущий реальный портфель (лайв/тестер) и дата; далее в конце каждой свечи
  • пересчёт линейного % и % с капитализацией (не сумма L1…L10).

Контроль ресурсов ПК (только предупреждение в лог, торговлю не блокирует):

Сравнивается свободная RAM и место на диске (каталог Engine\) с оценкой нагрузки:

— число активных логик;

— точки истории портфелей в памяти;

— вкладки скринера и уникальные индикаторы;

— размер файлов портфелей на диске (в т.ч. крупнейший файл).

Проверка: при старте, после смены логик, загрузки JSON, сохранения портфелей;

повтор того же предупреждения — не чаще ~5 минут.

Если ресурсов мало — в лог пишется, сколько нужно RAM/диска и сколько не хватает;

уменьшите число активных логик, вкладок скринера или объём истории портфелей.

Вкладка «Металогики» (сразу после «Логики»):

Вверху — «Металогика включена», «Инверсия (покупка ↔ продажа)», кнопка «Включить металогику»,

PnlSMA (вкл. и длина); под PnlSMA — разделитель; ниже — SMA, Stoch, ATR, LinReg, MACD.

Пока «Металогика включена», но по каждой активной логике не накопилось окно PnlSMA

(длина — «Общепортфельный PnlSMA: длина», по одной точке equity на свечу) — входы как без металогики:

Volume поровну, приоритет Max positions L1…L10, без переворота по PnlSMA.

После готовности PnlSMA: Volume делится между логиками с Op и PnlSMA>0 пропорционально PnlSMA;

логики с PnlSMA≤0 не торгуют (PnlSMA=0 тоже исключается).

«Инверсия (покупка ↔ продажа)» — только когда металогика уже действует (PnlSMA готов):

разрешить PnlSMA<0 с Buy↔Sell и Op↔Cl (как инверсия логики); доля Volume по |PnlSMA|. По умолчанию выкл.

Сумма объёмов новых входов на свече не превышает Volume (первая вкладка).

Max positions: при нехватке слотов — приоритет логик с большим PnlSMA.

Если металогика выключена — как раньше: каждая логика отдельно, Volume поровну, L1…L10.

Общепортфельные SMA, Stoch, ATR, LinReg, MACD, PnlSMA — один набор параметров;

расчёт по кривой equity каждой логики L1…L10 и по сумме (файлы _LogicPortfolio_Ln, _MetaAggregate).

Отдельных «ЛП1…ЛП10» в параметрах нет — настройки общие для всех портфельных серий.

При включённом мета-индикаторе робот считает его по серии equity портфеля

(если на вкладке графика нет соответствующего индикатора) и пишет значения в файл

портфеля (v2, поле meta) и Engine\{имя}_MetaAggregate.txt для общепортфельных.

JSON-снимок v2 включает meta в истории портфелей и блок AggregateMetaPortfolio.

PnlSMA (Приведённая SMA): pnlSmaAvg = mean(E_i − E_start) по окну; pnlSmaLast = E_end − E_start;

металогика использует pnlSmaAvg; по умолчанию включён PnlSMA, длина окна — 24 свечи.

Вкладка «Stopper» (общепортфельная страховка по сумме L1…L10):

Equity = сумма кривых портфелей всех логик (realized + unrealized MultiLogic_Ln).

«Общепортфельный stop-loss / take-profit: включён» — по умолчанию выкл.

SL/TP по ATR суммарной equity: ATR_port = mean(|Δequity|) за N точек агрегата L1…L10 (N — «Длина ATR портфеля для SL/TP»).

Ref — «Предыдущая сумма» (кнопка «Обновить сумму портфеля SL/TP» или авто при первом цикле).

SL: equity ≤ ref − K_sl×ATR_port (K_sl — «Общепортфельный stop-loss кратность ATR», по умолчанию 2).

TP: equity ≥ ref + K_tp×ATR_port (K_tp — «Общепортфельный take-profit кратность ATR», по умолчанию 3).

Проверка — один раз за цикл «общей свечи» скринера (страница 1) или на каждой свече монитора (страница 2).

При срабатывании — закрытие всех позиций робота; для SL и TP отдельно: «Отключено», «Останавливать полностью» (Regime=Off),

«Останавливать и возобновлять при достижении текущей суммы в фейк-торговле» (реальные заявки не выставляются, сигналы в L1…L10;

возобновление реальной торговли при equity фейк-портфеля ≥ текущей суммы на момент срабатывания +0,1%).

Тех. поля: текущая/ref сумма, ATR портфеля, пороги SL/TP по ATR — только отображение.

После SL/TP ref = equity после закрытия всех позиций.

«Stopper: таймфрейм монитора» («Как основной TF» / Min1…Min15): «Как основной TF» — проверка на общей свече страницы 1;

  • страница 2 скринера удаляется; Min1…Min15 — вторая страница (оранжевая «2») с «Stopper: бумага монитора» (по умолчанию SBER).

HTML-отчёт: журнал Stopper с K×ATR, ATR и порогом equity.

0) Отключение логики (только в самом начале строки, до AND/OR/&&/||):

Disabled(true) — логика отключена, индикаторы не создаются
Disabled(false) — явно включена (то же, что без префикса)
Disable(true/false) — синоним Disabled
  • Без префикса Disabled — логика включена.
  • Нельзя внутри скобок фрагмента или после AND/OR/&&/|| — только в начале, без внешних скобок.
  • Примеры:
Disabled(true) SMA(100) Op[Ab] Cl[Bl]
Disabled(false) (SMA(100) Op[Ab]) AND (ATR(14;Gr=3%;Lb=5) Op[GrOk] Cl[-])

0a) Strict — строгость отбора (необязательно, в начале строки, после Disabled, до Regime):

  • Strict(@Strict) — значение из параметра «Strict (строгость)» на вкладке «Логики» (как @LR).
  • Strict(4) — явное 1…5 для этой строки (не зависит от параметра робота).
  • Без Strict(…) — используется параметр робота. Подробно — раздел «6b) Strict».
Пример: Strict: (@Strict) Regime(LinReg;L=@LR;…) (SMA(100) Op[Ab] Cl[Bl]) …
0b) Regime — режим наклона LinReg (необязательно, только в начале строки, после Disabled и Strict):
Regime(LinReg;L=@LR;Dev=2;SlopeLb=5;SlopeDead=0.05%;OnFlip=Close;Entry=MatchSide;OnFlat=Close)
  • Entry=MatchSide — Buy при наклоне вверх, Sell вниз; во флэте входа нет
  • Entry=FlatOnly — вход только во флэте (|Δ| ≤ SlopeDead); при выходе из флэта — RegimeFlip
  • OnFlip=Close — закрыть, если наклон против Side (MatchSide) или вышли из флэта (FlatOnly)
  • OnFlip=Flip — переворот: закрыть и сразу открыть по новому наклону (Buy при sign>0, Sell при sign<0);
  • во флэте (sign=0) — только закрытие, если сработал OnFlat=Close
  • OnFlat=Close — MatchSide: закрыть позицию во флэте; OnFlat=Keep — не закрывать во флэте (FlatOnly)
  • SlopeLb=5 — сравнить центр канала (серия 1) с центром N свечей назад
  • SlopeDead=0.05% — мёртвая зона |Δцентра|; внутри — флэт
Regime(Auto;OnFlip=Close;Entry=MatchSide) — параметры из первого LinReg с Op[SlopeUp/Dn] в строке
Op[SlopeUp] / Op[SlopeDn] в атоме LinReg — альтернатива Entry=MatchSide внутри AND
L1/L3: Regime(…;SlopeLb=3;OnFlip=Close;Entry=MatchSide) — Buy при наклоне вверх, Sell вниз.
L2/L4: Regime(…;SlopeLb=3;SlopeDead=0.05%;OnFlip=Close;Entry=FlatOnly) — вход только во флэте; выход при тренде.
  • SlopeDead и OnFlat в defaults не заданы — можно добавить вручную (см. параметры Regime выше).

1) Формат v2 — Op(…) и Cl(…) с блоками Long / Short:

  • Strict(@Strict) Regime(LinReg;L=@LR;Dev=2;SlopeLb=3;Entry=MatchSide)
Op: (Long(SMA(100)(Ab) AND LinReg(@LR;Dev=2)(AbUp) AND ATR(14;Gr=3%;Lb=5)(GrOk)))
Cl: (Long(SMA(100)(Bl) AND LinReg(@LR;Dev=2)(BlLo)) OnFlip(Close)) SL[@SL] TP[@TP] Note(trend)
  • Индикатор: Ind(параметры)(условие) — 1-я скобка: настройка, 2-я: сигнал (Ab, K>=10, ValUp, &&, ||, !).
Общие фильтры до Long/Short: Op: (ATR(14;Gr=3%;Lb=5)(GrOk) Long(…)) — один экземпляр ATR на графике.
  • Long приоритетнее Short, если оба истинны. Buy/Sell — синонимы Long/Short.
  • OnFlip в Cl/Op: Close | Flip | Open; параметры (;Entry=…;OnFlat=…) приоритетнее Regime(…).
  • Параметры управления (@…): @Strict, @LR, @SL/@TP (×ATR на позицию; 0 — выкл.).

1a) Логические операторы внутри Long(…) / Short(…) и общих предикатов:

1a) Логические операторы (NOT, &&, || — в стиле JavaScript)

A) Между фрагментами (скобочные атомы в одной строке «Логика N»):

  • OR — слово OR с пробелами, или оператор ||
  • AND — слово AND с пробелами, или оператор &&
  • NOT — префикс NOT, NOT- или ! перед фрагментом (атом или (подвыражение))
  • Приоритет (как в JS): сначала ||, затем &&, затем NOT/!, затем атом.
Скобки: (…) группируют подвыражение: NOT ((A) && (B)).
  • Примеры между фрагментами:
(SMA(100) Op[Ab] Cl[Bl]) && (ATR(14;Gr=3%;Lb=5) Op[GrOk] Cl[-])
(SMA(100) Op[Ab] Cl[Bl]) || (SMA(100) Side[S] Op[Bl] Cl[Ab])
NOT LinReg(@LR;Dev=2) Op[AbUp] Cl[BlLo]
!LinReg(@LR;Dev=2) Op[AbUp] Cl[BlLo] — то же, что NOT
(SMA(100) Op[Ab] Cl[Bl]) && NOT (LinReg(@LR;Dev=2) Op[AbUp] Cl[BlLo])

B) NOT у всего атома (перед именем индикатора, после Side/SL/TP/Note):

  • NOT / ! / NOT- перед SMA, LinReg, Stoch… инвертирует логический результат Op и Cl
  • (true↔false). Side[S]/Side[L] и Buy/Sell НЕ меняются — только «сработало / не сработало».
Пример: NOT LinReg(@LR;Dev=2) Op[AbUp] Cl[BlLo]
  • → если AbUp без NOT был бы true — вход по этому атому блокируется (ни Buy, ни Sell);
  • → если AbUp без NOT был бы false — условие Op становится true (вход возможен, сторона как в Side).
  • Cl инвертируется так же: Cl сработал ↔ Cl не сработал. Это не шорт вместо лонга.
  • Переворот Buy↔Sell и Op↔Cl — «Инверсия логики» или «Инверсия» для PnlSMA<0 при металогике; NOT не меняет Side.
Можно комбинировать с NOT между фрагментами: NOT (NOT LinReg(...) Op[AbUp]) — двойное отрицание.
C) Внутри Op[…] и Cl[…] (только сигналы ЭТОГО индикатора атома):
  • ! или NOT перед кодом сигнала — отрицание одного условия
  • && или AND — оба условия должны выполниться
  • || или OR — достаточно одного условия
  • Приоритет внутри Op/Cl: ||, затем &&, затем !/NOT, затем код сигнала.
Скобки внутри Op/Cl: Op[: (AbUp||AbLo) && !BlLo].
Op[Ab && ValUp] — close > SMA и значение SMA выросло к прошлой свече
Op[Bl && ValDn] — close < SMA и SMA уменьшилась
Cl[Bl || ValDn] — выход: цена ниже SMA или SMA падает
  • Примеры внутри Op/Cl (LinReg):
Op[AbUp] — close выше верхней линии
Op[!AbUp] Op[NOT AbUp] — close НЕ выше верхней
Op[AbUp && AbLo] — выше верхней И выше нижней (сильный тренд в канале)
Op[AbUp || AbLo] — выше верхней ИЛИ выше нижней
Op[: (AbUp||AbLo) && !BlUp] — (выше верхней или нижней) и не ниже верхней
Cl[BlLo] Cl[!BlLo] — выход по обычному или инвертированному BlLo
Примеры: (Stoch): Op[K>=55 && K<=80] Cl[K<=45 || K>=90]
Примеры: (CCI): Op[CCI>=100] Cl[CCI<=-100] Op[!CCI>=100]

D) Что НЕ смешивается:

Внутри одного Op[…] нельзя ссылаться на другой индикатор (MACD>Sig только в атоме MACD).
Disabled(…) — только в самом начале строки логики, не внутри Op/Cl.
SL/TP не поддерживают &&/|| — только один уровень SL[…] TP[…] на атом.

E) Таблица записи (эквиваленты):

  • ИЛИ между фрагментами OR ||
  • И между фрагментами AND &&
  • НЕ фрагмент NOT NOT- !
  • НЕ сигнал в Op/Cl NOT AbUp !AbUp

2) Параметры индикатора (общие правила):

позиционно: Stoch(14-3-3), SMA(100), ATR(14;+3%;@5)
именованно: Stoch(K1=14,K2=3,D=3,Lmin=55,Smax=45), SMA(L=100,Src=Close)

3) Условия во 2-й скобке Ind(…)(условие):

  • Ab — close выше линии; Bl — close ниже; GrOk — ATR вырос (фильтр);
  • K>=55 / K<=45 — стохастик; K>=Lmin / K<=Smax — пороги из параметров;
CCI>=100 / CCI<=-100 — CCI (пороги Lmin/Smax в строке);
  • Macd>Sig / Macd<Sig — линия MACD выше/ниже сигнальной;
  • AbUp / BlUp — close выше/ниже верхней линии LinReg.
  • ValUp / ValDn / Chg>=0.05% — динамика значения индикатора.
  • Составные условия с !, NOT, &&, || — см. раздел «1a)».

4) SL / TP (на уровне строки, после Cl):

SL[@SL] TP[@TP] — кратность ATR из параметров «SL/TP позиции кратность ATR» (0 — выкл.).
Явно: SL[2%], SL[1.5ATR], TP[2R] (R — кратность к расстоянию SL).
  • В одной логике несколько атомов: самый жёсткий SL, самый дальний TP.
ATR для SL[…ATR]/TP[…ATR] — из первого ATR-атома той же логики.

5) Слоты «Логика 1…10»: при изменении любой строки робот перечитывает все 10 параметров.

  • Непустая включённая логика — её индикаторы добавляются в общий набор.
  • Одинаковые индикатор с теми же параметрами в разных логиках — один экземпляр на графике.
  • Другие параметры — отдельный индикатор. Пустые/Disabled логики индикаторы не добавляют.
  • Номера индикаторов: 101 … 1099 (общий пул, без дублей).

6a) Стандартные логики UT, UCT, L1…L4 (кнопка «Установить логики по умолчанию», вкладка «Логики»)

  • В FINRESP-калькуляторе в каталоге первой идёт TBC (test counter-bokovik), затем UT, UCT, L5 и L1…L4.
  • Записывает четыре строки в «Логика 1…4», очищает «Логика 5…10»; индикаторы на графике — сразу.
  • Смысл набора L1…L4: на базе TrendMultiIndicator+CCI — два лонга (тренд и боковик) и два шорта (тренд и боковик).
  • L1/L3 — Regime MatchSide (вход по наклону LinReg); L2/L4 — Regime FlatOnly (вход во флэте, Stoch от границ).
  • При Regime=On на одной свече могут сработать L1+L2 (лонг) и L3+L4 (шорт) — разные фильтры, не дубли.
SL[@SL] TP[@TP] — кратность ATR из параметров вкладки «Логики»; 0 = без SL/TP по позиции.

TBC — test counter-bokovik (лонг)

Строка: (@Strict) Regime(LinReg;L=@LR;Dev=2;SlopeLb=3;SlopeDead=0.05%;OnFlip=Close) Op(Long(SMA(100)(Bl) AND Stoch(14-3-3;Lmin=90;Smax=10)(K<=10) AND MACD(12,26,9)(Macd<Sig))) Cl(Long(SMA(100)(Ab) AND Stoch(14-3-3;Lmin=90;Smax=10)(K>=90) AND MACD(12,26,9)(Macd>Sig))) SL[@SL] TP[@TP] Note(test-counter-bokovik)
  • Только лонг; контртренд-боковик: вход ниже SMA (Bl), Stoch перепродан + MACD<Sig; выход выше SMA (Ab), Stoch перекуплен + MACD>Sig.
  • Regime со SlopeDead=0.05%, без Entry=FlatOnly (в отличие от L2).

UT — Universal Trend (лонг + шорт, тренд)

Строка: (@Strict) Regime(LinReg;L=@LR;SlopeLb=3;OnFlip=Close) Op(Long(SMA(100)(Ab) AND LinReg(@LR;K=@K)(AbLinK))) Cl(Long(SMA(100)(Bl) AND LinReg(@LR;K=@K;Anchor=Open;Drift=RegDrift)(BlRegK) OnFlip(Close))) Op(Short(SMA(100)(Bl) AND LinReg(@LR;K=@K)(BlLinK))) Cl(Short(SMA(100)(Ab) AND LinReg(@LR;K=@K;Anchor=Open;Drift=RegDrift)(AbRegK) OnFlip(Close))) SL[@SL] TP[@TP] Note(universal-trend)
  • Тренд всегда — зеркало UCT: пробой канала LinReg по направлению движения, без SlopeDead и Entry=FlatOnly.
  • @K — ширина канала в ×ATR (по умолчанию 2); длина @LR — в свечах.
  • Op — лонг AbLinK (close ≥ mid+K×ATR) + SMA(Ab); шорт BlLinK + SMA(Bl).
  • Cl — RegDrift от якоря открытия: лонг BlRegK + SMA(Bl); шорт AbRegK + SMA(Ab).
  • OnFlip(Close) — выход при смене наклона LinReg.

UCT — Universal Counter Trend (лонг + шорт, контртренд)

Строка: (@Strict) Regime(LinReg;L=@LR;SlopeLb=3;OnFlip=Close) Op(Long(SMA(100)(Bl) AND LinReg(@LR;K=@K)(BlLinK))) Cl(Long(SMA(100)(Ab) AND LinReg(@LR;K=@K;Anchor=Open;Drift=RegDrift)(AbRegK) OnFlip(Close))) Op(Short(SMA(100)(Ab) AND LinReg(@LR;K=@K)(AbLinK))) Cl(Short(SMA(100)(Bl) AND LinReg(@LR;K=@K;Anchor=Open;Drift=RegDrift)(BlRegK) OnFlip(Close))) SL[@SL] TP[@TP] Note(universal-counter-trend)
  • Контртренд всегда — без SlopeDead и без Entry=FlatOnly; боковик/тренд для входа не фильтруются.
  • @K — ширина канала LinReg в ×ATR (по умолчанию 2): нижняя/верхняя граница = центр ± K×ATR.
  • Op — канал на текущем баре: лонг BlLinK (close ≤ mid−K×ATR) + SMA(Bl); шорт AbLinK + SMA(Ab).
  • Cl — якорь открытия + RegDrift = βopen×баров в позиции: лонг AbRegK (close ≥ midopen+RegDrift+K×ATR) + SMA(Ab); шорт BlRegK + SMA(Bl).
  • OnFlip(Close) — принудительный выход при смене наклона LinReg (SlopeLb из Regime).

L1 — lon-trend (Лонг, тренд)

Strict: (@Strict) Regime(LinReg;L=@LR;Dev=2;SlopeLb=3;OnFlip=Close;Entry=MatchSide) Op(Long(SMA(100)(Ab) AND LinReg(@LR;Dev=2)(AbUp) AND ATR(14;Gr=3%;Lb=5)(GrOk) AND CCI(20;Lmin=100;Smax=-100)(CCI>=100) AND MACD(12,26,9)(Macd>Sig))) Cl(Long(SMA(100)(Bl) AND LinReg(@LR;Dev=2)(BlLo) AND CCI(20;Lmin=100;Smax=-100)(CCI<=-100) AND MACD(12,26,9)(Macd<Sig)) OnFlip(Close)) SL[@SL] TP[@TP] Note(lon-trend)
  • Состав:
  • Regime MatchSide — Buy при наклоне LinReg вверх; во флэте входов нет.
• Op(Long(…)) — SMA(100)(Ab), LinReg(@LR;Dev=2)(AbUp), ATR(GrOk), CCI, MACD.
• Cl(Long(…)) OnFlip(Close) — выход когда все Cl-условия атомов (AND, как v1 Op[]/Cl[]); ATR Cl[-] только в Op.
•: ATR(14;Gr=3%;Lb=5)(GrOk) — общий фильтр волатильности.
  • @LR — параметр «Длина линейной регрессии».

L2 — lon-bokovik (Лонг, боковик)

Strict: (@Strict) Regime(LinReg;L=@LR;Dev=2;SlopeLb=3;SlopeDead=0.05%;OnFlip=Close;Entry=FlatOnly) Op(Long(SMA(100)(Ab) AND Stoch(14-3-3;Lmin=90;Smax=10)(K<=10) AND ATR(14;Gr=3%;Lb=5)(GrOk) AND MACD(12,26,9)(Macd>Sig))) Cl(Long(SMA(100)(Bl) AND Stoch(14-3-3;Lmin=90;Smax=10)(K>=90) AND MACD(12,26,9)(Macd<Sig)) OnFlip(Close)) SL[@SL] TP[@TP] Note(lon-bokovik)
  • Состав:
  • Regime FlatOnly+SlopeDead — вход только во флэте LinReg.
• Op(Long(…)) — SMA(100)(Ab), Stoch(K<=10), MACD>Sig.
  • Cl(Long(…)) OnFlip(Close) — выход Stoch/MACD и при выходе из флэта.
•: ATR(GrOk) — фильтр волатильности.

L3 — short-trend (Шорт, тренд)

Strict: (@Strict) Regime(LinReg;L=@LR;Dev=2;SlopeLb=3;OnFlip=Close;Entry=MatchSide) Op(Short(SMA(100)(Bl) AND LinReg(@LR;Dev=2)(BlLo) AND ATR(14;Gr=3%;Lb=5)(GrOk) AND CCI(20;Lmin=100;Smax=-100)(CCI<=-100) AND MACD(12,26,9)(Macd<Sig))) Cl(Short(SMA(100)(Ab) AND LinReg(@LR;Dev=2)(AbUp) AND CCI(20;Lmin=100;Smax=-100)(CCI>=100) AND MACD(12,26,9)(Macd>Sig)) OnFlip(Close)) SL[@SL] TP[@TP] Note(short-trend)
  • Состав:
  • Regime MatchSide — Op(Short(…)) вместо Side[S].
• Short: (SMA(100)(Bl) AND LinReg(BlLo) …) — медвежьи условия в Short-блоке.
  • Cl(Short(…)) OnFlip(Close) — зеркало L1.
•: ATR(GrOk) — фильтр волатильности.

L4 — short-bokovik (Шорт, боковик)

Strict: (@Strict) Regime(LinReg;L=@LR;Dev=2;SlopeLb=3;SlopeDead=0.05%;OnFlip=Close;Entry=FlatOnly) Op(Short(SMA(100)(Bl) AND Stoch(14-3-3;Lmin=90;Smax=10)(K>=90) AND ATR(14;Gr=3%;Lb=5)(GrOk) AND MACD(12,26,9)(Macd<Sig))) Cl(Short(SMA(100)(Ab) AND Stoch(14-3-3;Lmin=90;Smax=10)(K<=10) AND MACD(12,26,9)(Macd>Sig)) OnFlip(Close)) SL[@SL] TP[@TP] Note(short-bokovik)
  • Состав:
  • Regime FlatOnly — Op(Short(…)) во флэте (зеркало L2).
• Short: (Stoch(K>=90) …) — вход у верхней границы Stoch.
  • Cl(Short(…)) OnFlip(Close).
•: MACD(Macd<Sig) — импульс вниз.

6b) Strict (строгость отбора) — параметр на вкладке «Логики», целое 1…5, по умолчанию 3.

  • В начале строки (после Disabled, до Regime): Strict(@Strict) или Strict(4).
  • @Strict — токен параметра робота (как @LR для длины LinReg); число 1…5 — фиксированная строгость логики.
  • Без префикса Strict(…) используется параметр «Strict (строгость)».
  • 1–2 — мягче (больше сделок); 4–5 — строже (меньше сделок); 3 — пороги в тексте без масштабирования.
  • В строке хранятся нейтральные пороги (как при strict=3); парсер масштабирует Lmin/Smax/Gr/SlopeDead/Dev
  • и числа в Op/Cl (CCI, K, RSI, MOM) при разборе. Кнопки ±1 меняют параметр и сразу перепарсивают логики.
Пример: Strict: (@Strict) Regime(LinReg;L=@LR;…) (SMA(100) Op[Ab] Cl[Bl]) …
  • Для одной логики жёстче остальных: Strict(5) … (остальные с Strict(@Strict) или без префикса).

7) СПРАВОЧНИК ИНДИКАТОРОВ — как записывать (полный набор)

--- Динамика значения индикатора (ValUp / Chg — для SMA, RSI, CCI, MACD, …) ---

Сравнивается основная серия индикатора (не close): SMA→линия SMA, MACD→линия MACD, Boll→середина полос, LinReg→центр.

Параметры в строке атома (только логика, не на график):

;Streak=2 — ValUp/ValDn: N свечей подряд роста/падения (или ValUp2 / ValDn3 в Op[…])

;ValDead=0.05% — минимальный |% изменения| за одну свечу для ValUp/ValDn

;ChgLb=2 — окно для Chg (по умолчанию 1; Chg2>=… — явно 2 свечи)

;ChgMin=0.1% — порог для Op[Chg>=ChgMin] / Cl[Chg<=ChgMin]
Сигналы: (в Op[…] и Cl[…], можно с && / ||):

ValUp / Rise — значение выросло к прошлой свече

ValDn / Fall — значение уменьшилось

ValUp2 / ValDn3 — рост/падение N свечей подряд

Chg>=0.05% — рост ≥ 0.05% за ChgLb свечей (Chg2>=0.1% — за 2 свечи)

Chg<=-0.05% — падение ≥ 0.05% за окно

Chg>=ChgMin — порог из ;ChgMin=… в атоме

LinReg: Op[SlopeUp/Dn] — наклон центра за SlopeLb с мёртвой зоной; ValUp — только vs 1 свеча.

Примеры:

L1 тренд: (SMA(100) Op[Ab && ValUp] Cl[Bl || ValDn]) AND …
L3 шорт: (SMA(100) Side[S] Op[Bl && ValDn] Cl[Ab || ValUp]) AND …
MACD: MACD(12,26,9) Op[Macd>Sig && ValUp] Cl[Macd<Sig || ValDn]
RSI %: RSI(14;ChgLb=2;ChgMin=0.5%) Op[RSI>=55 && Chg>=ChgMin] Cl[RSI<=45]
---: SMA (Simple Moving Average) ---

Имя в строке: SMA

Параметры (на график):

SMA(100) — длина 100, источник Close
SMA(L=100) — то же, явно
SMA(L=100,Src=Close) — длина и источник (Close)

Сигналы:

Op[Ab] — close > SMA (лонг); Cl[Bl] — close < SMA (выход из лонга)
Op[Ab && ValUp] — close > SMA и линия SMA растёт (см. раздел ValUp/Chg)
Side[S] Op[Bl && ValDn] Cl[Ab || ValUp] — шорт: цена ниже SMA и SMA падает

Примеры:

Тренд: SMA(100) Op[Ab && ValUp] Cl[Bl || ValDn] SL[2%] TP[6%] Note(trend)
Антитренд: SMA(100) Side[S] Op[Bl && ValDn] Cl[Ab || ValUp] Note(counter-trend)
С ATR: (SMA(100) Op[Ab && ValUp] Cl[Bl]) AND (ATR(14;Gr=3%;Lb=5) Op[GrOk] Cl[-])
---: Stochastic (Stoch) ---
Имя в строке: Stoch (или Stochastic)

Параметры индикатора (P1-P2-P3 на график):

Stoch(14-3-3) — периоды 14, 3, 3
Stoch(14,3,3) — через запятую
Stoch(K1=14,K2=3,D=3) — именованно

Пороги сигнала (только в строке логики, не на график):

;Lmin=55;Smax=45 или ;L=55;S=45

Сигналы:

Op[K>=55] / Op[K>=Lmin] — %K выше порога (лонг)
Cl[K<=45] / Cl[K<=Smax] — %K ниже порога (выход)

Примеры:

Тренд: Stoch(14-3-3;Lmin=55;Smax=45) Op[K>=55] Cl[K<=45] Note(trend)
Антитренд: Stoch(14-3-3;Lmin=55;Smax=45) Side[S] Op[K<=45] Cl[K>=55] Note(counter-trend)
---: ATR (фильтр роста волатильности, без направления) ---

Имя в строке: ATR

Параметры индикатора:

ATR(14) — период 14
ATR(L=14) — то же

Пороги роста (в строке логики):

;Gr=3%;Lb=5 — ATR вырос ≥ 3% относительно значения 5 свечей назад

;+3%@5 — краткая запись Gr/Lb

Сигналы:

Op[GrOk] или Op[+3%@5] — фильтр «волатильность выросла»
Cl[-] — отдельного выхода по ATR нет

Примеры:

Фильтр: ATR(14;Gr=3%;Lb=5) Op[GrOk] Cl[-] Note(volatility-filter)
С SMA: (SMA(100) Op[Ab] Cl[Bl]) AND (ATR(14;Gr=3%;Lb=5) Op[GrOk] Cl[-])

--- Linear Regression (LinReg) ---

Имя в строке: LinReg (или LR, LinearRegression)

Параметры индикатора:

LinReg(@LR) — длина из параметра «Длина линейной регрессии», Dev=2
LinReg(@LR;Dev=2) — длина из параметра, отклонение границ
LinReg(50;Dev=2) — явная длина 50 (не из параметра)
LinReg(L=@LR,Dev=2) — именованно, длина из параметра
LinReg(@LR;K=@K) — канал ±K×ATR вокруг центра (Universal Counter Trend)
LinReg(@LR;K=@K;Anchor=Open;Drift=RegDrift) — выход: midopen + β×Δt ± K×ATR

Сигналы канала K×ATR (FINRESP / Universal Counter Trend):

Op[BlLinK] — close ≤ mid − K×ATR (лонг, отскок снизу)
Op[AbLinK] — close ≥ mid + K×ATR (шорт, отскок сверху)
Cl[AbRegK] — close ≥ midopen + RegDrift + K×ATR (выход лонга)
Cl[BlRegK] — close ≤ midopen − RegDrift − K×ATR (выход шорта)
RegDrift = βopen × баров в позиции; β — приращение центра LinReg за 1 бар на входе

Сигналы Dev×σ (как TrendMultiIndicatorScreener — канал LinReg):

Op[AbUp] — close выше верхней линии (лонг)
Cl[BlUp] — close ниже верхней (выход)
Side[S] Op[BlLo] Cl[AbLo] — шорт от нижней линии

Пороги наклона (только в строке логики, не на график):

;SlopeLb=3 — сравнить центр канала с центром 3 свечи назад

;SlopeDead=0 — мёртвая зона |Δцентра| (по умолчанию 0)

Op[SlopeUp] — центр LinReg растёт (Δ > SlopeDead); лонг-режим
Op[SlopeDn] — центр падает (Δ < -SlopeDead); шорт-режим
Cl[-] с Op[SlopeUp/Dn] — атом только фильтр режима, без своего выхода

Примеры:

Тренд: LinReg(@LR;Dev=2) Op[AbUp] Cl[BlUp] SL[2%] TP[6%] Note(trend)
Антитренд: LinReg(@LR;Dev=2) Side[S] Op[BlLo] Cl[AbLo] Note(counter-trend)

--- MACD ---

Имя в строке: MACD (или Macd)

Параметры индикатора:

MACD(12,26,9) — fast, slow, signal
MACD(Fast=12,Slow=26,Signal=9)

Сигналы:

Op[Macd>Sig] — линия MACD выше сигнальной (лонг)
Cl[Macd<Sig] — MACD ниже сигнальной (выход / шорт)

Примеры:

Тренд: MACD(12,26,9) Op[Macd>Sig] Cl[Macd<Sig] SL[2%] TP[6%] Note(trend)
Антитренд: MACD(12,26,9) Side[S] Op[Macd<Sig] Cl[Macd>Sig] Note(counter-trend)
---: CCI (Commodity Channel Index) ---

Имя в строке: CCI

Параметры индикатора (на график):

CCI(20) — длина 20, источник Typical (H+L+C)/3
CCI(L=20) — то же, явно
CCI(L=20,Src=Typical) — длина и точка свечи (Typical, Close, Open, High, Low, Median)
CCI(20;Src=Close) — через точку с запятой

Пороги сигнала (только в строке логики, не на график):

;Lmin=100;Smax=-100 или ;L=100;S=-100 — классические уровни перекупленности/перепроданности

Сигналы:

Op[CCI>=100] / Op[CCI>=Lmin] — CCI выше порога (лонг от силы / контртренд — по смыслу стратегии)
Cl[CCI<=-100] / Cl[CCI<=Smax] — CCI ниже порога (выход)
Op[CCI<=-100] Side[S] — шорт при CCI ниже −100 (контртренд)

Примеры:

Контртренд: CCI(20;Lmin=100;Smax=-100) Side[S] Op[CCI>=100] Cl[CCI<=-100] Note(CCI-fade-short)

Не включён в логики по умолчанию и не входит в металогики (только строки «Логика 1…10»).

RSI(14) — длина 14, источник Close
RSI(L=14) — то же, явно

Пороги сигнала (только в строке логики, не на график):

;Lmin=55;Smax=45 или ;L=55;S=45

Сигналы:

Op[RSI>=55] / Op[RSI>=Lmin] — RSI выше порога (лонг / сила)
Cl[RSI<=45] / Cl[RSI<=Smax] — RSI ниже порога (выход)

Bollinger(20) — длина 20, Dev=2 по умолчанию
Bollinger(20;Dev=2) — длина и отклонение
Bollinger(L=100,Dev=2) — именованно

Op[Ab] — close выше верхней полосы; Cl[Bl] — close ниже нижней

Пробой: Bollinger(20;Dev=2) Op[Ab] Cl[Bl] Note(trend-boll-break)

8) Торговля (Regime = On)

  • На каждой закрытой свече каждой вкладки скринера проверяются все не-Disabled логики.
Вход: срабатывает составное выражение по Op[…] (||/OR, &&/AND, NOT/! между фрагментами).
Выход: по Cl[…]: (те же операторы внутри Cl[…] и NOT у атома); Cl[-] — атом не задаёт направленный выход.
  • Side[S] — шорт (Sell), иначе лонг (Buy). Позиция: сигнал MultiLogic_L1 … MultiLogic_L10.
  • «Инверсия логики (покупка ↔ продажа)» (вкладка «Логики», по умолчанию выкл.): как в TrendMultiIndicatorScreener —
bull↔bear: вход по Cl[…] вместо Op[…], выход по Op[…] вместо Cl[…], Buy↔Sell (NOT в строке не меняется).
  • Volume — общий объём; при нескольких входах на одной свече делится поровну между логиками.
  • Max positions (all tabs) — лимит открытых позиций робота на скринере.
  • Нехватка слотов: входят логики с меньшим номером (1 раньше 10), остальные пропускаются.
SL/TP: на закрытии свечи проверяется close vs уровни из SL[…]/TP[…] строки логики позиции.
Пробой SL — закрытие с сигналом …_SL; пробой TP — …_TP; иначе выход по Cl[…].

9) Stopper — общепортфельный SL/TP по ATR (вкладка «Stopper»)

  • Мониторинг: суммарная equity L1…L10 (PnL портфелей логик; до сделок — депозит).
  • Проверка SL/TP — один раз за цикл «общей свечи» скринера (после всех вкладок страницы 1),
  • либо на каждой свече монитора (страница 2), если выбран отдельный TF.
  • ATR_port — среднее |Δequity| за N точек агрегата L1…L10 (N = «Длина ATR портфеля для SL/TP»).
  • ref — база из «Предыдущая сумма» (кнопка «Обновить сумму портфеля SL/TP» или авто при первом цикле).
  • SL: equity ≤ ref − K_sl×ATR_port (K_sl = «Общепортфельный stop-loss кратность ATR», по умолчанию 2).
  • TP: equity ≥ ref + K_tp×ATR_port (K_tp = «Общепортфельный take-profit кратность ATR», по умолчанию 3).
  • Галочки «включён» управляют только активацией; K=0 — порог выключен.
  • Тех. поля: ATR портфеля, пороги SL/TP по ATR — только для отображения.
  • После срабатывания ref обновляется на equity после закрытия всех позиций.
  • HTML-отчёт: журнал Stopper с колонками K×ATR, ATR, порог equity.
