Операции * (свёртка), # / /# (покомпонентно), / (деление), () (композиция); сдвиг — δδ(k)

pp oo hh ll vv — рынок (OHLCV) bb ee — сделки (покупки / продажи) ww ss tt … — индикаторы / производные (1/N; 1/N; …) — коэффициенты δδ(k) — дискретная δ, единица на позиции k

1. Обозначения

Многочлен — вектор коэффициентов. Позиция k — бар k (0 — самый старый).

pp = (p₀; p₁; p₂; …; p_T)     Close
oo = (o₀; o₁; …)               Open
hh, ll   (h₀; …), (l₀; …)         High, Low
vv = (v₀; v₁; v₂; …)           Volume
bb = (b₀; b₁; b₂; …)           объём покупок (Buy) на баре
ee = (e₀; e₁; e₂; …)           объём продаж (sEll) на баре
ЗаписьСмысл
pp oo hh ll vv рынок OHLCV (фуксия)
bb, ee объёмы сделок: покупки и продажи (оранжевый); на баре k — b_k, e_k
ww, ss, tt, … индикатор / производный (синий)
(a₀; a₁; …) коэффициенты — тело многочлена (янтарный жирный)
aa # bb покомпонентно умножить: (aa#bb)k = ak·bk
aa /# bb покомпонентно разделить: (aa/#bb)k = ak / bk
aa * bb произведение: (aa*bb)k = Σi+j=k ai·bj
aa / bb деление многочленов (не покомпонентное; не /#)
aa ± bb покомпонентно сложить / вычесть
δδ(k) дискретная δ: (0; …; 0; 1; 0; …) — единица только на позиции k, остальные нули (аналог δ-функции)
pp * δδ(n) сдвиг на n баров → (0; …; 0; p₀; p₁; …)
δδ(1) сдвиг на 1 бар; то же, что (0; 1; 0; …)
pp(qq) композиция (подстановка qq в pp): (pp(qq))n = Σi=0n pi·(qq(*i))n, где qq(*0)=(1;0;0;…), qq(*i)=qq*qq(*i−1)
pp(8) подстановка числа вместо индекса: одно значение Σk≥0 pk·8k (порождающий многочлен в точке 8); на баре 8 — коэффициент p8 или выборка pp * δδ(8)
ff(1) ff — доходы/расходы по сделкам на баре; ff(1)=Σk fk — общий финрез (сумма всех коэффициентов)

δδ(k) — короткая запись многочлена «0…010…0». Свёртки: δδ(i) * δδ(j) = δδ(i+j). Примеры: δδ(2) = (0; 0; 1; 0; …), δδ(3) = (0; 0; 0; 1; 0; …).

Что такое δδ (часто читают «дельта-дельта», не «дубль-эль»).

Это две греческие буквы δ (дельта) подряд — наша запись дискретной δ-функции (импульса Кронекера) по номеру бара.

  • δδ(k) — многочлен-вектор: на позиции k стоит 1, на всех остальных — 0. В таблице коэффициенты такого вектора показаны янтарным цветом: (0; …; 0; 1; 0; …).
  • Сама буква δδ в тексте — фиолетовая (как в легенде сверху). Не путать с коэффициентами в скобках — они янтарные, но это просто числа внутри многочлена.
  • Зачем: сдвиг ряда на n баров назад: «взять вчерашнее закрытие» = pp * δδ(1). Разность цен: pp − pp * δδ(1).
  • Не латинская ll (Low), не операция умножения — только «единичный импульс на баре k».
δδ(0) = (1; 0; 0; 0; …)     «только бар 0»
δδ(1) = (0; 1; 0; 0; …)     «только бар 1» — сдвиг на 1
δδ(3) = (0; 0; 0; 1; 0; …)   «только бар 3»

На баре t: коэффициент (pp * δδ(n))t = pt−n   (цена n баров назад)
Хвост. При ss = pp * ww коэффициенты s₀…sN−2 — неполная свёртка. Пока не используем; берём k ≥ N−1. 
ss = pp * ww          весь ряд индикатора сразу
s_k = k-й коэффициент ss

Композиция

Запись pp(qq) — не покомпонентное умножение, а подстановка внутреннего многочлена (композиция рядов). Запись pp(8) — подстановка скаляра.

qq(*0) = (1; 0; 0; …)
qq(*i) = qq * qq(*i−1)     i ≥ 1

(pp(qq))n = Σi=0n pi · (qq(*i))n

pp(8) = Σk≥0 pk · 8k              одно число (значение ряда в точке 8)

pp(δδ(8)) = pp * δδ(8)          сдвиг/выборка: на позиции 8 — p₀, далее p₁, …

Пример: если ss = pp * ww_SMA — это свёртка; pp(ss) — другая операция (композиция коэффициентных рядов). Не путать с #.

2. Простейшие индикаторы

SMA(N)

ww_SMA = (1/N; 1/N; …; 1/N)          длина N

ss_SMA = pp * ww_SMA

s_k = (p_k + … + p_{k−N+1}) / N    при k ≥ N−1

EMA(α)

ww_EMA = (α; α(1−α); α(1−α)²; …)

ss_EMA = pp * ww_EMA

WMA(N)

S = N(N+1)/2
ww_WMA = (1/S; 2/S; 3/S; …; N/S)

ss_WMA = pp * ww_WMA

Типичная цена (CCI)

uu = (1/3; 1/3; 1/3; …)

tt = (hh + ll + pp) # uu
ss_TP = tt * ww_SMA

VWAP (сессия, MultiLogic)

Средняя цена, взвешенная по объёму, с сбросом в начале календарного дня. На графике Prime — индикатор VWAP без параметров.

uu = (1/3; 1/3; 1/3; …)

tt = (hh + ll + pp) # uu          типичная цена HLC/3,  t_k = (h_k + l_k + p_k) / 3
qq = tt # vv                        q_k = t_k · v_k

ww_ONE = (1; 1; 1; …)                 единичные веса (нарастающая сумма)

nn = qq * ww_ONE                  n_k = q₀ + … + q_k   (внутри сессии, с нуля дня)
fv = vv * ww_ONE                  f_k = v₀ + … + v_k   (накопленный объём; не путать с ff сделок)

ss_VWAP = nn /# fv              vwap_k = n_k / f_k

Покомпонентное деление — /#, не /: (nn/#fv)k = nk / fk. Веса не фиксированы заранее — они зависят от объёмов v_j внутри сессии (это не один многочлен ww как у SMA).

Сброс сессии. Свёртка с ww_ONE накапливает с начала ряда. Для VWAP на баре k суммируем только бары текущего календарного дня (в OsEngine — TimeStart.Date). Формально: перед суммой обнуляем qq и vv вне сегодняшней сессии или пересчитываем накопление с индекса начала дня.

На баре k внутри сессии (с индекса s дня):

vwap_k = (t_s·v_s + … + t_k·v_k) / (v_s + … + v_k)

Лонг Op[Ab]:  p_k > vwap_k
Лонг Cl[Bl]:  p_k < vwap_k

4. LinReg — линейная регрессия и канал

В MultiLogic / калькуляторе: LinReg(@LR; Dev=2) или LinReg(@LR; K=@K) (канал ±K×ATR для UT/UCT). Длина окна N = @LR — в свечах. На баре k берём последние N закрытий p_{k−N+1}, …, p_k.

4.1. Окно и оси (как в движке)

Внутри окна номер бара j = 0 … N−1: j = 0 — самая старая свеча в окне, j = N−1 — текущая. Цена y_j = p_{k−N+1+j}.

x_j = j                          горизонтальная ось (время внутри окна)
y_j = p_{k−N+1+j}                вертикаль — Close

Среднее по окну: ȳ_k = (p_{k−N+1} + … + p_k) / N = (pp * ww_SMA)k
ww_SMA = (1/N; 1/N; …; 1/N)   длина N

4.2. Наклон (первый коэффициент регрессии, slope)

Наклон прямой МНК (метод наименьших квадратов) по точкам (j, y_j):

β_k = [ N·Σ_j (j·y_j) − (Σ_j j)·(Σ_j y_j) ] / [ N·Σ_j j² − (Σ_j j)² ]

В движке: β_k = (N·sumXY − sumX·sumY) / (N·sumXX − sumX·sumX)

Через свёртку с антисимметричными весами (центр окна в (N−1)/2):

x̄ = (N − 1) / 2
S_xx = Σ_{j=0}^{N−1} (j − x̄)²

ww_SLOPE = ( w₀; w₁; …; w_{N−1} ),   w_j = (j − x̄) / S_xx

β_k = (pp * ww_SLOPE)k = Σ_{j=0}^{N−1} w_j · p_{k−N+1+j}

Пример N = 5: x̄ = 2, S_xx = 10, ww_SLOPE = (−0,2; −0,1; 0; 0,1; 0,2). Положительный β_k — цена в окне в среднем растёт слева направо.

Знак тренда в Regime(LinReg; …): сравнивают β на текущем баре с β SlopeLb баров назад (не отдельный индикатор, а разность двух наклонов).

4.3. Уровень линии (второй коэффициент — intercept и значение на текущем баре)

Регрессия ŷ_j = α_k + β_k · j. Intercept (значение прямой на j = 0, начало окна):

α_k = ȳ_k − β_k · x̄ = ȳ_k − β_k · (N − 1) / 2

Через многочлены:
α_k = (pp * ww_SMA)k − (pp * ww_SLOPE)k · (N − 1) / 2

Центр канала LinReg на текущем баре — значение прямой на j = N−1 (конец окна). В коде это linregMid / c1:

mid_k = α_k + β_k · (N − 1) = ȳ_k + β_k · (N − 1) / 2

ss_mid = (pp * ww_SMA) + (pp * ww_SLOPE) # ( (N−1)/2; (N−1)/2; … )

Здесь # — покомпонентное умножение на константу (N−1)/2 в каждой позиции (скаляр размазан по ряду). Сложение aa + bb — тоже покомпонентное.

Значение на начале окна (для справки): α_k = mid_k − β_k · (N − 1).

4.4. Полосы ±Dev (стандартное отклонение остатков)

Остаток на точке j: e_j = y_j − (α_k + β_k · j). В многочленах на баре k для каждого j нужна разность «цена минус подгонка»; компактная запись через подгонку на всём окне:

ss_fit = … α_k + β_k·j на окне …     (подогнанная прямая по N точкам)
ss_res = pp − ss_fit                  остатки (покомпонентно в окне)

σ_k = sqrt( (1/N) · Σ_j e_j² )
    = sqrt( ( (ss_res # ss_res) * ww_SMA )k )

Границы канала при Dev = 2 (как в L1…L5, TBC):

ss_up = ss_mid + (2; 2; …) # ss_σ
ss_lo = ss_mid − (2; 2; …) # ss_σ

AbUp / BlLo в Op/Cl: сравнение pp с ss_up или ss_lo на баре k

4.5. Канал ±K×ATR (UT / UCT)

Центр тот же ss_mid; ширина от ATR (см. §5), не от σ остатков:

ss_ATR = rr * ww_SMA          из §5, длина ATR по параметру индикатора

ss_up = ss_mid + (K; K; …) # ss_ATR
ss_lo = ss_mid − (K; K; …) # ss_ATR

K = @K в калькуляторе — множитель ATR (по умолчанию 2), не число свечей.

4.6. Сводка LinReg только через *, #, /#, ±

ww_SMA   = (1/N; …; 1/N)
ww_SLOPE = ( (0−x̄)/S_xx; …; (N−1−x̄)/S_xx ),  x̄ = (N−1)/2

β   = pp * ww_SLOPE                         наклон
ȳ   = pp * ww_SMA                          среднее Close в окне
mid = ȳ + β # ((N−1)/2; …)              центр LinReg на текущем баре
σ   = sqrt( ((pp − fit) # (pp − fit)) * ww_SMA )   Dev-канал
up/lo = mid ± Dev # σ   или   mid ± K # ss_ATR

5. Волатильность

Диапазон

rr = hh − ll

ATR(N)

ss_ATR = rr * ww_SMA

Bollinger(N, k)

ss_mid = pp * ww_SMA

6. Осцилляторы

CCI(N)

ss_TP = tt * ww_SMA
dd = tt − ss_TP
CCI_k ≈ d_k / (0.015 · mean(|dd| на окне))

RSI(N)

dd = pp − pp * δδ(1)
gg = max(dd, 0)
ll_rsi = max(−dd, 0)     не путать с ll Low

ss_g = gg * ww_SMA
ss_l = ll_rsi * ww_SMA
kk = ss_g /# ss_l

RSI_k ≈ 100 − 100 · (1 + k_k)−1     на баре k

Stochastic %K(N)

%K_k ≈ (p_k − min_N(l_k)) / (max_N(h_k) − min_N(l_k))

MACD(f, s, sig)

ss_fast = pp * ww_EMA(f)
ss_slow = pp * ww_EMA(s)

ss_macd = ss_fast − ss_slow
ss_sig  = ss_macd * ww_EMA(sig)
ss_hist = ss_macd − ss_sig

7. Сделки и финансовый результат

Многочлен сделок — пара объёмов на каждом баре: bb (покупки) и ee (продажи). Буквы: Buy и sEll; цвет оранжевый — отдельно от свечей pp…vv, но это тоже «сырые» рыночные величины (кластер / лента), не индикатор.

В OsEngine на кластере: VolumeBuy, VolumeSell; чистый поток на баре: VolumeDelta ≈ b_k − e_k.

Чистый поток и позиция

xx = bb − ee                    x_k = b_k − e_k   (дельта объёма на баре)

ww_ONE = (1; 1; 1; …)

zz = xx * ww_ONE                  z_k = x₀ + … + x_k   (накопленная позиция в лотах)

xx и zz — производные от сделок (синий), как индикаторы; bb и ee — исходные объёмы.

Доходы и расходы по сделкам (ff)

ff — многочлен денежного потока на баре (доходы минус расходы от исполнений). Приближение «вся сделка по p_k»: покупка уменьшает кэш, продажа увеличивает.

ff = pp # (ee − bb)              f_k = p_k · (e_k − b_k)

ff(1) = Σk≥0 fk = f₀ + f₁ + … + fT     общий финрез (сумма коэффициентов; подстановка 1)

ff(1) = (ff * ww_ONE)T            то же на последнем баре T

Запись ff(1) — как у pp(8): порождающий ряд Σ fk xk в точке x=1. Это не полный PnL с открытой позицией: кэш и стоимость позиции раздельно. Для удержания — mark-to-market ниже.

Финансовый результат (mark-to-market)

Приращение цены и позиция на начало бара (удерживали z_{k−1}, цена изменилась на p_k − p_{k−1}):

Δpp = pp − pp * δδ(1)            (Δpp)_k = p_k − p_{k−1}

zz_open = zz * δδ(1)                z_{k−1} на позиции k

rr_mtm = zz_open # Δpp             r_k = z_{k−1} · (p_k − p_{k−1})

RR = rr_mtm * ww_ONE              R_k = r₀ + … + r_k   (накопленный финрез)
Ограничения. (1) Одна цена pp на бар — для тиковых сделок нужны свои цены исполнения. (2) zz — суммарный лот; шорт и лонг в одной формуле — знаковая позиция. (3) Комиссии, проскальзывание, своп — отдельные слагаемые, не выводятся из pp и bb/ee без дополнительных многочленов. (4) Реализованная прибыль по паре «вход–выход» — рекуррентная логика FIFO/LIFO, не одна свёртка с фиксированным ядром.

Стратегия SMA-разности как многочлены сделок

Покупка и продажа задаются на каждой свече как коэффициенты многочленов bb и ee. Объём сделки равен модулю разности между ценой и SMA: выше SMA — покупка, ниже SMA — продажа. Для реальной торговли добавьте множитель лота mm и округление до шага инструмента; ниже mm можно считать единичным многочленом.

ww_SMA = (1/N; …; 1/N)
ss_SMA = pp * ww_SMA                     s_k = SMA_N(p)_k
dd = pp − ss_SMA                         d_k = p_k − s_k

ii_B = H(dd)                              i^B_k = 1, если d_k > 0, иначе 0
ii_E = H(−dd)                             i^E_k = 1, если d_k < 0, иначе 0

bb_SMA = mm # dd # ii_B                  b_k = m_k · (p_k − s_k), если p_k > s_k
ee_SMA = mm # (−dd) # ii_E               e_k = m_k · (s_k − p_k), если p_k < s_k

xx_SMA = bb_SMA − ee_SMA                  чистый поток сделок на каждой свече
zz_SMA = xx_SMA * ww_ONE                  z_k = x₀ + … + x_k

ff_SMA = pp # (ee_SMA − bb_SMA)            денежный поток каждой свечи
ff_SMA(1) = Σ f_k                         сумма всех доходов/расходов по сделкам

CC_SMA = ff_SMA * ww_ONE                  C_k = f₀ + … + f_k
EQ_SMA = CC_SMA + (zz_SMA # pp)          mark-to-market кривая результата
F_T = (EQ_SMA)_T

То же как сумма всех свечей:
F_T = Σ_{k=0..T} p_k·(e_k − b_k) + p_T·Σ_{k=0..T}(b_k − e_k)

8. Связь с MultiLogic

На баре k — k-й коэффициент ss. Условия сравнивают p_k или s_k с порогом.

oo и pp — разные многочлены. Тело: (pp − oo). Close > Open: p_k > o_k.

9. Сводка

Имя:     pp = (p₀; p₁; …; p_T)
