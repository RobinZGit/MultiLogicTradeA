1. Индикатор расчёта

При нажатии Рассчитать вверху страницы появляется синяя полоса «Идёт расчёт»: после заголовка — полоса-ползунок (заполнение слева направо), справа — текст этапа; процент в конце строки этапа (… — 45%):

  • подбор фьючерсов MOEX (если есть фьючерсы) — начало диапазона загрузки;
  • загрузка цен MOEX / кэш IndexedDB — 0–33% по числу загруженных инструментов;
  • расчёт FINRESP — 33–66% по свечам (240/480 свечей в тексте); без Stopper этот этап идёт до 99%;
  • Stopper портфеля (если SL/TP включены) — 66–99%, текст Stopper портфеля · N/M свечей;
  • во время расчёта — не выше 99%; 100% и закрытие полосы только после полного завершения (графики и результат).
  • справа в полосе — кнопка Остановить: прерывает загрузку MOEX и расчёт; показываются FINRESP и графики до последней досчитанной свечи (правая граница окна сдвигается). При остановке во время загрузки — по уже загруженным инструментам.

На время расчёта тема страницы слегка меняется, все поля и кнопки формы заблокированы (кроме «Остановить»). После завершения блокировка снимается.

2. Запуск

Калькулятор — MultiLogic_FinrespCalculator.html и скрипты в одной папке:

  • MultiLogic_FinrespCalculator.engine.js — расчёт FINRESP и симуляция
  • MultiLogic_FinrespCalculator.charts.js — интерактивные графики по инструментам (свечи, pan/zoom)
  • MultiLogic_FinrespCalculator.live.js — T-Bank и блок «Реальная торговля»

Для MOEX нужен http://, не file://.

  • Локально: serve-calculator.ps1 → http://127.0.0.1:8765/MultiLogic_FinrespCalculator.html
  • GitHub Pages: онлайн-версия
  • Перенос: скопируйте папку FinrespCalculator целиком

При открытии через file:// появляется жёлтое предупреждение — свечи MOEX не загрузятся.

Настройки формы (режим кошелька, даты, объёмы, логики L1…L5, галочки, live-инструменты в «Заявка»/«Стакан», @OBT) сохраняются в localStorage браузера под ключом multilogic.finresp.config.v1 и восстанавливаются при следующем открытии. Токен T-Bank — отдельно, зашифрован.

5.1. Параметры @ (основной блок)

В синей карточке «Основной расчёт», сразу под выбором логики — поля подстановки в строку Op/Cl. Символ @ — плейсхолдер; при «Рассчитать» и «Применить» числа подставляются в SL[@SL], LinReg(@LR;K=@K), Strict(@Strict) и т.д.

ПолеНазначение
@SL — стоп, ×ATRКратность ATR для stop-loss позиции; 0 — выкл. Не абсолютная цена
@TP — тейк, ×ATRTake-profit позиции; кратность ATR
Длина ATR, свечейПериод ATR в барах (свечах) — общий для @SL/@TP, @SmaCorridor и @K
@SmaCorridor — SMA, ×ATRШирина коридора ±K×ATR вокруг SMA (логики SMA-эталон); не в свечах
@LR — LinReg, свечейДлина линейной регрессии в барах (свечах), не в ATR. Кнопка «Оптимизировать»
@K — LinReg, ×ATRШирина канала: по умолчанию 2×ATR (не свечи, не абсолютная цена). UT/UCT: LinReg(@LR;K=@K). Кнопка «Оптимизировать»
@Strict — строгостьStrict(@Strict) в начале строки; 1…5. Кнопка «Оптимизировать»

5.2. Параметры @@ (управление портфелем)

Блок «Параметры управления» — действуют на все логики независимо от строки Op/Cl.

ПолеНазначение
Реверс сторон (Long ↔ Short) @@ReverseSidesСвоп сторон стратегии: меняются местами все сигналы Op/Cl. То есть Op(Long)↔Op(Short) и Cl(Long)↔Cl(Short) (включая условия, SL/TP и «flip-выход» через Op противоположной стороны). Практически: любая «лонговая» логика превращается в её шорт-эквивалент и наоборот. В live при переключении — сначала закрытие всех позиций по рынку
Реверс сигналов (инверсия условий) @@ReverseSignalsИнвертирует сами условия внутри Op/Cl, не меняя «сторону блока» Long/Short. Примеры: Ab↔Bl, AbUp↔BlLo, AbLinK↔BlLinK, AbRegK↔BlRegK, а также сравнения индикаторов. Для осцилляторов 0…100 (Stoch K, RSI) инверсия делает «низ↔верх»: K<=20 → K>=80, K>=80 → K<=20 (правило \(v→100-v\) + инверсия знака). Это как раз позволяет получать тренд/контртренд‑вариант из одной строки
Максимальное количество позиций @@MaxPosМакс. число «слотов» позиций по портфелю (по умолчанию 10). Плечо = @@MaxPos × Volume% / 100. Лимит gross = депозит × @@MaxPos × Volume%
@@SL / @@TP / @@ATRПортфельный Stopper по суммарной equity

Сводка @@ReverseSides=… @@MaxPos=… плечо=… под полями @@. Volume% (10% по умолчанию) — в «Доп. параметрах» §9.1.

Совместное включение (XOR): инверсия уровней (ReverseSignals) применяется, когда включён ровно один из двух реверсов — Signals или Sides, но не оба. Своп сторон (ReverseSides) — когда включён Sides. Порядок: сначала инверсия уровней, затем своп сторон. Таблица для TotStoch-логик из FTS:

  • без реверсов → FTS (лонг, контртренд);
  • только ReverseSignals → FTT (лонг, тренд);
  • только ReverseSides → FTS_S (шорт, контртренд);
  • оба реверса → FTT_S (шорт, тренд) — «обратная» логика, эквивалент нативной FTT_S.

9.1.1. Портфельный лимит gross (несколько инструментов и live)

При расчёте по нескольким бумагам и в live / песочнице действует общий потолок суммарной экспозиции (|лонг| + |шорт| по всем тикерам в рублях):

лимит ₽ = депозит × Max positions × Volume%

  • Volume% — размер типичной новой позиции на одну бумагу (лот одной сделки), поле в «Доп. параметрах».
  • @@MaxPos — сколько таких «слотов» можно набрать суммарно по портфелю (§5.2).
  • Если открытие на следующем инструменте превысит лимит — объём обрезается или сделка не проходит (в песочнице — сообщение «Портфельный лимит…»).
  • Тот же лимит используется в бэктесте по нескольким инструментам и при reconcile робота по FINRESP.

9.5. Используемые индикаторы

Чекбоксы SMA, ATR, Stoch, LinReg, Bollinger, VWAP, CCI, MACD, Momentum. Отключённые удаляются из Op/Cl перед расчётом. SL/TP позиции не зависят от этого списка.

Оптимизировать (индикаторы) — перебор непустых комбинаций, остаётся набор с лучшим FINRESP.

9.6. Каталог логик — все встроенные стратегии по умолчанию

В блоке «Логики» (каталог) перечислены готовые строки Op/Cl/Regime — те же, что в роботе MultiLogic. Параметры @SL, @TP, @LR, @K, @Strict, @SmaCorridor, @CmaLen, @CmaPow подставляются из основного блока при «Рассчитать» и «Применить». Маркер @OBT в строку добавляется вручную (фильтр по стакану — только live).

Общие принципы

  • Regime(LinReg;…) — режим рынка по наклону линейной регрессии за @LR свечей. Entry=MatchSide — вход только по стороне тренда; Entry=FlatOnly — только в боковике (мертвая зона наклона SlopeDead). OnFlip=Close — при смене режима закрытие по условию Cl с суффиксом OnFlip(Close).
  • Op(…) — открытие позиции; Cl(…) — закрытие. Long / Short — направление.
  • Ab / Bl у индикатора — цена выше / ниже линии (close относительно SMA, полосы Bollinger, канала LinReg и т.д.).
  • LinReg — канал: центр регрессии ± Dev (в L1…L5) или ± K×ATR (в UT/UCT). Сигналы AbUp/BlLo — пробой верхней/нижней границы; AbLinK/BlLinK — цена относительно канала K×ATR; AbRegK/BlRegK — с дрейфом якоря Drift=RegDrift.
  • SL[@SL] / TP[@TP] — стоп и тейк позиции (доля от цены входа или ×ATR — по значению полей).
  • Strict(@Strict) — строгость совпадения всех условий в скобках (0…1).
  • Несколько логик в списке «Логика» — стек приоритета: на баре сначала проверяется верхняя, затем следующая; выход и SL/TP — у логики, открывшей позицию. SMA-логики с непрерывным объёмом в стек с L1…L5 не смешиваются — оставьте одну SMA или одну L-логику.

RND — случайные сделки (тест)

Назначение: контрольный шум — проверка расчёта FINRESP и инфраструктуры без рыночной логики.

Принцип: на каждом баре независимо с вероятностью 12% сигнал Rand(P=12%) на long и отдельно на short. Фиксированные SL[1%] и TP[5%]. Regime не используется.

TBC — test counter-bokovik (лонг)

Назначение: контртрендовый лонг в боковике — покупка на перепроданности у нижней границы диапазона.

Regime: LinReg с SlopeDead=0.05%, Entry=FlatOnly — только когда рынок «без тренда».

Вход long: close ниже SMA(100) и Stoch %K ≤ 10 и MACD линия ниже сигнальной.

Выход long: close выше SMA(100) и Stoch %K ≥ 90 и MACD выше сигнальной. SL/TP — из @SL/@TP.

UT — Universal Trend

Назначение: торговля по тренду в обе стороны — long на росте, short на падении, с каналом LinReg ±K×ATR.

Regime: LinReg без мёртвой зоны; смена режима может закрывать через OnFlip(Close) на Cl.

Long: вход — цена выше SMA(100) и пробой верхней границы канала LinReg(@LR;K=@K) (AbLinK). Выход — цена ниже SMA и регрессионный канал с дрейфом якоря на Open (BlRegK) + OnFlip.

Short: зеркально — вход BlLinK под SMA, выход AbRegK над SMA. Ширина канала — поле @K (×ATR), длина — @LR.

UCT — Universal Counter Trend

Назначение: контртренд — long от нижней границы канала при общем контексте, short от верхней; те же LinReg K×ATR и RegDrift, но сигналы инвертированы относительно UT.

Long: вход под SMA + BlLinK; выход над SMA + AbRegK OnFlip.

Short: вход над SMA + AbLinK; выход под SMA + BlRegK OnFlip.

L5 — LmaxTrend (лонг + short, максимальный набор фильтров)

Назначение: трендовая стратегия с полным стеком индикаторов — для сравнения «тяжёлого» варианта L1/L3.

Regime: трендовый (Entry=MatchSide).

Long: SMA(100) сверху + LinReg AbUp + Bollinger AbUp + VWAP Ab + ATR «рост волатильности» (GrOk) + Stoch ≥ 80 + CCI ≥ 100 + Momentum > 0 + MACD > сигнальная. Выход — зеркальные «медвежьи» условия (BlLo, Bl по полосам, Stoch ≤ 20, CCI ≤ −100, MOM < 0, MACD < sig) + OnFlip.

Short: симметрично для нисходящего тренда (BlLo, нижние полосы, Stoch ≤ 20, CCI ≤ −100 и т.д.).

L1 — лонг, тренд

Regime: тренд, вход по стороне тренда.

Вход long: цена выше SMA(100), LinReg AbUp (±Dev=2), ATR GrOk (рост ≥ 3% за 5 баров), CCI ≥ 100, MACD > сигнальная.

Выход: цена ниже SMA, LinReg BlLo, CCI ≤ −100, MACD < сигнальная, OnFlip(Close).

L2 — лонг, боковик

Regime: боковик (SlopeDead, FlatOnly).

Вход long: цена выше SMA, Stoch %K ≤ 10 (откат вверх от перепроданности в диапазоне), ATR GrOk, MACD > sig.

Выход: цена ниже SMA, Stoch ≥ 90, MACD < sig, OnFlip.

L3 — short, тренд

Зеркало L1 для short: вход под SMA + LinReg BlLo + ATR + CCI ≤ −100 + MACD < sig; выход над SMA + AbUp + CCI ≥ 100 + MACD > sig, OnFlip.

L4 — short, боковик

Зеркало L2 для short: Regime боковик; вход short при цене ниже SMA и Stoch ≥ 90; выход при цене выше SMA и Stoch ≤ 10, OnFlip.

sma_below — «Ниже SMA», объём |Close−SMA|

Назначение: эталонная SMA-стратегия без дискретных Op/Cl — непрерывный набор/сброс объёма.

Принцип: строка SMA(100;Vol)(Bl) — пока close ниже SMA(100), позиция long с объёмом, пропорциональным расстоянию |Close−SMA| (чем дальше ниже — тем больше лот в рамках настроек объёма). Выход при возврате цены к SMA или по SL/TP. Маркеры на графике — при переходе через ноль и по стопам.

sma_above — «Выше SMA», объём |Close−SMA|

То же, но условие (Ab) — удержание long, пока цена выше SMA; объём растёт с отдалением вверх от средней.

sma_corridor_trend — SMA-эталон, тренд, коридор ATR

Принцип: SMA(100;Spread=@SmaCorridor)(Trend) — вокруг SMA строится коридор ±@SmaCorridor×ATR. В режиме Trend набор объёма при движении по направлению выхода за коридор (импульс в сторону тренда относительно SMA). Непрерывная позиция, SL/TP из полей.

sma_corridor_anti — SMA-эталон, анти-тренд, коридор ATR

(Anti) — зеркальная логика: объём на откатах против импульса за границу коридора (контртренд относительно SMA+ATR-полосы).

FTS — Фьючерс: TotStoch + Stoch (лонг), контртренд 20↔80

Идея: покупаем «внизу пилы» и продаём «вверху» на боковике фьючерсов, но с подтверждением от общей (тотальной) цены выбранных инструментов.

TotP: на каждом баре строится тотальная цена (TotOpen/TotHigh/TotLow/TotClose) как среднее OHLC по выбранным инструментам (как «Средняя цена выбранных инструментов (Close)» на equity-блоке, но для всех OHLC).

TotStoch: стохастик (14-3-3) считается по TotOHLC, т.е. %K от тотальной цены.

Вход long: TotStoch.K ≤ 20 и одновременно Stoch.K ≤ 20 инструмента.

Выход long: TotStoch.K ≥ 80 и одновременно Stoch.K ≥ 80 инструмента. SL/TP — по полям @SL/@TP.

FTT — Фьючерс: TotStoch + Stoch (лонг), тренд 80↔20

Идея: трендовый вариант на той же тотальной базе: покупаем, когда рынок «разогнан вверх» одновременно по корзине и по инструменту, выходим при «развороте вниз».

Вход long: TotStoch.K ≥ 80 и одновременно Stoch.K ≥ 80 инструмента.

Выход long: TotStoch.K ≤ 20 и одновременно Stoch.K ≤ 20 инструмента. SL/TP — по полям @SL/@TP.

FTS_S — Фьючерс: TotStoch + Stoch (шорт), контртренд 80↔20

Зеркальная версия FTS для short: продаём «вверху пилы», выходим «внизу».

Вход short: TotStoch.K ≥ 80 и одновременно Stoch.K ≥ 80 инструмента.

Выход short: TotStoch.K ≤ 20 и одновременно Stoch.K ≤ 20 инструмента. SL/TP — по полям @SL/@TP.

FTT_S — Фьючерс: TotStoch + Stoch (шорт), тренд 20↔80

Зеркальная версия FTT для short: продаём «внизу» по стохастикам, закрываемся «вверху».

Вход short: TotStoch.K ≤ 20 и одновременно Stoch.K ≤ 20 инструмента.

Выход short: TotStoch.K ≥ 80 и одновременно Stoch.K ≥ 80 инструмента. SL/TP — по полям @SL/@TP.

CML — CM long (CMA + LinReg на входе)

Назначение: кастомная средняя CMA с показателем степени @CmaPow и длиной @CmaLen — только лонг.

Вход long: цена выше CMA и LinReg AbUp (пробой верхней границы регрессии, Dev=2). Regime в строке нет — фильтр только по CMA+LinReg на Op.

Выход: цена ниже CMA, OnFlip(Close) — закрытие при пересечении CMA вниз.

CMS — CM short (CMA + LinReg на входе)

Зеркало CML для short: вход — цена ниже CMA + LinReg BlLo; выход — цена выше CMA, OnFlip(Close).

Для UT, UCT, L1…L5 канал LinReg на графике — фиолетовые линии, если LinReg включён в «Индикаторы» и входит в выбранную логику. Текстовые поля Op/Cl/Regime в каталоге редактируются как в роботе; Применить пересчитывает с подстановкой @-параметров.

10.1. Графики по инструментам (calc-chart)

После «Рассчитать» (или в live-сессии при обновлении графиков) для каждого инструмента строится отдельный блок с тикером и FINRESP по этой бумаге.

Цена — свечи

  • Отображаются японские свечи (Open / High / Low / Close) по барам выбранного окна.
  • Зелёная свеча — close ≥ open; красная — close < open.
  • Данные OHLC берутся из загруженных свечей MOEX/T-Bank; при расчёте они также попадают в строки симуляции.

Масштаб и прокрутка

График интерактивный — на ПК и на планшете (мышь + touch):

ДействиеЭффект
Колёсико мыши над графикомМасштаб по времени (zoom in/out) вокруг точки под курсором
Перетаскивание (зажатая ЛКМ)Сдвиг видимого окна влево/вправо по барам
Свайп одним пальцем (планшет/сенсор)То же, что drag — сдвиг по времени; страница при этом не прокручивается (touch-action: none на области графика)
Pinch (два пальца — разведение/сведение)Масштаб вокруг середины между пальцами (zoom in при разведении, out при сведении)
Двойной клик или двойной tapВернуть отображение всего периода окна

В подсказке на графике показывается, сколько баров видно из общего числа (если включён zoom).

Линии индикаторов

На графике рисуются линии индикаторов, которые используются в выбранных логиках (галочки в блоке «Логика»), а не только у «активной» логики в стеке:

  • SMA — оранжевая пунктирная; для SMA-коридора — также верхняя/нижняя граница коридора
  • LinReg — фиолетовая (центр и полосы ±Dev или ±K×ATR для BlLinK / AbRegK и т.п.)
  • Bollinger — голубая (средняя и верх/низ)
  • VWAP — зелёная пунктирная
  • FINRESP инструмента — чёрная линия на правой шкале (₽), после первого △/▲. Значение: eq − eq₀, где eq₀ — equity на баре первого входа/выхода (eq = cash + qty×цена). Пока по бумаге не было сделок — линия не рисуется. В правом верхнем углу графика — подпись FINRESP … ₽ по видимому окну (с учётом zoom/pan), в рамке; обновляется при масштабировании

Если индикатор отключён в «Индикаторы» или не входит ни в одну выбранную логику — линия не рисуется. Под каждым графиком — легенда: образец линии (цвет и пунктир) и название. На самой картинке графика (в т.ч. при «Копировать график») — тикер инструмента слева.

Копирование графика

Справа от тикера инструмента — кнопка «Копировать график». Копирует текущий видимый фрагмент (с учётом масштаба и сдвига) в буфер обмена как PNG. Нужен HTTPS или localhost и разрешение браузера на доступ к буферу.

Маркеры входа и выхода

Маркеры ставятся при смене позиции в симуляции (в т.ч. flip long↔short и SL с повторным входом на одном баре). У △/▲ — подпись: первая строка — имя логики (L1, UT, SMA…), вторая — фрагмент из парсера (Op(Long(...)), Cl(Short(...)), SL[…×ATR] и т.п.). При наведении — полная подсказка.

МаркерЗначение
△ остриём вниз над high свечиВход — зелёный long, красный short; подпись: логика + Op(…)
▲ остриём вверх под low свечиВыход — Cl(…), Op flip, SL/TP или Sma → flat

Размер △/▲ зависит от ширины свечи на экране. Подписи скрываются при очень плотном масштабе (узкие свечи); приблизьте участок колёсиком.

У SMA-логик с непрерывным набором/сбросом объёма маркеры — при переходе позиции через ноль и при SL/TP.

FINRESP, стопы и портфель

  • Бумага без сделок (нет △/▲) — линия FINRESP и подпись в углу не показываются; портфельный stopper не пересчитывает её хвост.
  • SL / TP по позиции — тонкая вертикальная (SL красная, TP зелёная); только на барах с △/▲.
  • SL / TP портфельного стоппера — жирнее, другой пунктир; метка события по портфелю на всех графиках.

Live-сессия (режим «Реальная торговля»)

После выбора кошелька «Реальная торговля T-Bank» графики по инструментам показывают окно с момента начала live-сессии (не весь исторический расчёт). FINRESP на mini-графике и в блоке результатов — от нуля сессии (baseline на первом баре сессии). Цветной фон: зелёный — песочница, розовый — реальная торговля. Вертикальные пунктиры — фактические заявки (синяя покупка, оранжевая продажа).

Диагностика

Блок «Тех. информация» внизу формы: после открытия графиков — строки chart.ТИКЕР: … (число входов/выходов, posStop, линия FINRESP). Скопируйте и пришлите при сбое отображения.

Файлы и версия

Интерактивная отрисовка — MultiLogic_FinrespCalculator.charts.js (рядом с engine.js и live.js). Версия страницы — бейдж pageVersion на форме (например 2026-06-14-marker-labels-v1). После обновления — Ctrl+Shift+R (скрипты с параметром ?v= в URL).

10.2. Equity портфеля (calc-chart-equity)

Отдельный блок ниже ползунков: кривая equity выбранного стека, справочные кривые по логикам каталога, портфельный стоппер. Формат — линия по equity, без свечей и без pan/zoom.

Над блоком — примечание: превращения FINRESP под каждым equity-графиком считаются за тот же таймфрейм, что поле «Таймфрейм» расчёта (1 мин … 1 день). Полоска баров Δ: зелёный — рост за свечу/период, розовый — просадка; на неактивных логиках — синие оттенки. Кнопка «Копировать график» — PNG кривой и полоски Δ.

Заявки в реальном времениТаблица активных и недавних заявок. В песочнице — «Заявки (фейк, симуляция)» с пометкой (фейк)

Не все тикеры MOEX доступны для торговли через API T-Bank (api_trade_available_flag). При «выбрать все акции» калькулятор предпочитает листинг MOEX (TQBR), проверяет статус до заявки и пропускает недоступные бумаги, продолжая торговлю по остальным. Токен должен иметь право на торговлю (не только чтение).

При live без предварительного «Рассчитать» или при расхождении окон свечей возможно предупреждение «FINRESP пустой» / windowSkipped — свечи и опрос продолжаются, торговля не останавливается; сузьте выбор или нажмите «Рассчитать».
