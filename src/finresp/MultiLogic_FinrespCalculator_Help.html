1. Индикатор расчёта

При нажатии Рассчитать вверху страницы появляется синяя полоса «Идёт расчёт»: после заголовка — полоса-ползунок (заполнение слева направо), справа — текст этапа; процент в конце строки этапа (… — 45%):

  • подбор фьючерсов MOEX (если есть фьючерсы) — начало диапазона загрузки;
  • загрузка цен MOEX / кэш IndexedDB — 0–33% по числу загруженных инструментов;
  • расчёт FINRESP — 33–66% по свечам (240/480 свечей в тексте); без Stopper этот этап идёт до 99%;
  • Stopper портфеля (если SL/TP включены) — 66–99%, текст Stopper портфеля · N/M свечей;
  • во время расчёта — не выше 99%; 100% и закрытие полосы только после полного завершения (графики и результат).
  • справа в полосе — кнопка Остановить: прерывает загрузку MOEX и расчёт; показываются FINRESP и графики до последней досчитанной свечи (правая граница окна сдвигается). При остановке во время загрузки — по уже загруженным инструментам.

На время расчёта тема страницы слегка меняется; большинство полей и кнопок заблокированы (кроме «Остановить» и блока выбора логик — см. §5). После завершения блокировка снимается. Длинный расчёт (много инструментов и логик) не должен «зависать» на 99%: этапы отдают управление UI через yieldToUi / cycleBeat (cycle-cooperative.js).

2. Запуск

Калькулятор — Angular-маршрут /finresp и скрипты в assets/finresp/:

  • MultiLogic_FinrespCalculator.engine.js — расчёт FINRESP и симуляция
  • MultiLogic_FinrespCalculator.stop-monitor.js — единая оценка стопов (портфель, @@PauseOnDrawdown, позиционные SL/TP) для live
  • MultiLogic_FinrespCalculator.charts.js — интерактивные графики по инструментам (свечи, pan/zoom)
  • MultiLogic_FinrespCalculator.live.js — T-Bank и блок «Реальная торговля»
  • bond-tbru-data.js, bond-tbru-fetch.js, bond-tbru-procedure.js — логика TBRU (портфель облигаций по составу фонда, §9.8)

Для MOEX нужен http://, не file://.

  • Локально: run-dev.bat → http://127.0.0.1:4200/finresp
  • GitHub Pages: онлайн-версия
  • Перенос: скопируйте папку FinrespCalculator целиком

При открытии через file:// появляется жёлтое предупреждение — свечи MOEX не загрузятся.

Настройки формы (режим кошелька, даты, объёмы, логики L1…L5, галочки, live-инструменты в «Заявка»/«Стакан», @OBT) сохраняются в localStorage браузера под ключом multilogic.finresp.config.v1 и восстанавливаются при следующем открытии. Токен T-Bank — отдельно, зашифрован.

@TP — тейк, ×ATRTake-profit позиции; кратность ATR
Длина ATR, свечейПериод ATR в барах (свечах) — общий для @SL/@TP, @SmaCorridor и @K

5.2. Параметры @@ (управление портфелем)

Верхняя часть блока «Параметры управления» — действуют на все логики независимо от строки Op/Cl.

ПолеНазначение
@@PauseOnDrawdown + @@DrawdownPctСтоп-лосс с восстановлением по портфелю (если не включён режим «по логикам» ниже): при просадке суммарной equity от пика на заданный % — закрыть все позиции и не открывать новые, пока модельная equity не вернётся к уровню пика. В расчёте — applyPauseOnDrawdown; в live — баннер, закрытие позиций (песочница — фейк, реал — рыночные заявки). Возобновление — при восстановлении модели
@@PauseOnDrawdownPerLogicВторая галочка в «Параметрах управления» и в toolbar live. При просадке изолированной equity одной логики (как на её mini-графике equity) — временно исключить только её из стека торговли; остальные логики продолжают работать. Позиции отключённой логики сводятся к модели (reconcile). В расчёте — applyPauseOnDrawdownPerLogic; в live — чип логики «отключена», состояние в config и сессии. Восстановление — когда модельная equity этой логики снова ≥ пика
@@PauseOnDrawdownPerInstrumentТретья галочка (взаимоисключающая с портфельной и «по логикам»). При просадке equity конкретного инструмента от пика на @@DrawdownPct % — остановить торговлю по этому тикеру; инструмент остаётся в выборе с пометкой «· пауза». Модельная (фейковая) equity инструмента отслеживается по полной симуляции; возобновление — когда модель ≥ пика. На графиках инструмента и в блоке equity — бледно-красная зона при отключении, вертикальные линии на границах, пунктир equity на момент паузы
Реверс сторон (Long ↔ Short) @@ReverseSidesСвоп сторон стратегии: меняются местами все сигналы Op/Cl. То есть Op(Long)↔Op(Short) и Cl(Long)↔Cl(Short) (включая условия, SL/TP и «flip-выход» через Op противоположной стороны). Практически: любая «лонговая» логика превращается в её шорт-эквивалент и наоборот. В live при переключении — сначала закрытие всех позиций по рынку
Реверс сигналов (инверсия условий) @@ReverseSignalsИнвертирует сами условия внутри Op/Cl, не меняя «сторону блока» Long/Short. Примеры: Ab↔Bl, AbUp↔BlLo, AbLinK↔BlLinK, AbRegK↔BlRegK, а также сравнения индикаторов. Для осцилляторов 0…100 (Stoch K, RSI) инверсия делает «низ↔верх»: K<=20 → K>=80, K>=80 → K<=20 (правило \(v→100-v\) + инверсия знака). Это как раз позволяет получать тренд/контртренд‑вариант из одной строки
Максимальное количество позиций @@MaxPosМакс. число «слотов» позиций по портфелю (по умолчанию 10). Плечо = @@MaxPos × Volume% / 100. Лимит gross = депозит × @@MaxPos × Volume%
@@SL / @@TP / @@ATRПортфельный Stopper по суммарной equity

Сводка @@ReverseSides=… @@MaxPos=… плечо=… под полями @@. Volume% (10% по умолчанию) — в «Доп. параметрах» §9.1.

Совместное включение (XOR): инверсия уровней (ReverseSignals) применяется, когда включён ровно один из двух реверсов — Signals или Sides, но не оба. Своп сторон (ReverseSides) — когда включён Sides. Порядок: сначала инверсия уровней, затем своп сторон. Таблица для фьючерсных логик FTS (CtgStoch):

  • без реверсов → FTS (лонг, контртренд);
  • только ReverseSignals → FTT (лонг, тренд);
  • только ReverseSides → FTS_S (шорт, контртренд);
  • оба реверса → FTT_S (шорт, тренд) — «обратная» логика, эквивалент нативной FTT_S.

9.1.1. Портфельный лимит gross (несколько инструментов и live)

При расчёте по нескольким бумагам и в live / песочнице действует общий потолок суммарной экспозиции (|лонг| + |шорт| по всем тикерам в рублях):

лимит ₽ = депозит × Max positions × Volume%

  • Volume% — размер типичной новой позиции на одну бумагу (лот одной сделки), поле в «Доп. параметрах».
  • @@MaxPos — сколько таких «слотов» можно набрать суммарно по портфелю (§5.2).
  • Если открытие на следующем инструменте превысит лимит — объём обрезается или сделка не проходит (в песочнице — сообщение «Портфельный лимит…»).
  • Тот же лимит используется в бэктесте по нескольким инструментам и при reconcile робота по FINRESP.

9.3.1. DSL стакана OB.* — парсинг и алгоритмы

В строке логики Op/Cl можно писать условия по снимку стакана так же, как условия по индикаторам. Формат атома:

OB.<Kind>(<params>)(<signal>)

Внутри Op(Long(…)) / Cl(Long(…)) атомы соединяются через AND (как SMA(100)(Ab) AND OB.Imb(…)(BuyOk)). Парсер распознаёт виды ob.imb, ob.spr, ob.depth (регистр и точка в OB.Imb не важны).

Таблица алгоритмов OB.*

АтомПараметрыСигналАлгоритм (на снимке стакана)Условие true

9.5. Используемые индикаторы

Чекбоксы SMA, ATR, ADX, Stoch, LinReg, Bollinger, VWAP, CCI, MACD, Momentum, TotStoch, CtgStoch. Отключённые удаляются из Op/Cl перед расчётом. SL/TP позиции не зависят от этого списка.

ADX — Average Directional Index (Wilder, период в скобках). Сигналы: TrOk — сила тренда ≥ Min (по умолчанию 25); WkOk — слабый тренд / флэт: ADX < Max (25). Дополнительно: DI+ / DI- — +DI выше / ниже −DI. Пример в трендовых логиках: ADX(14;Min=25)(TrOk) сразу после ATR(14;Gr=3%;Lb=5)(GrOk); в контртрендовых и боковике — ADX(14;Max=25)(WkOk).

ADX по умолчанию включён (галочка в списке индикаторов). Во встроенных L1…L5, UT, UCT, TBC и SMA-коридорах атом ADX уже в строке каталога. При открытии страницы ensureDefaultLogicLines подтягивает свежие строки из кода, если у L1…L5/UT/UCT/TBC есть ATR(…)(GrOk) без ADX или у боковика (L2, L4, TBC) ошибочно стоит TrOk вместо WkOk. При расчёте движок при необходимости дописывает ADX после ATR в Op-атомах (ensureAdxInLogicLineString, режим TrOk/WkOk по типу логики). Загрузка JSON-протокола/конфига: если в строках есть ADX(…), галочка ADX принудительно включается.

Составные индикаторы: кнопка + над списком — конструктор формул из встроенных индикаторов и pp; кнопка i. — справка по конструктору. Новый индикатор подсвечивается фиолетовым, включается галочкой и попадает в «Добавить индикатор» на графике как Имя().

TotStoch — стохастик по тотальной цене корзины (средняя OHLC всех выбранных инструментов на каждом баре). Работает на любом рынке, если атом есть в строке логики.

CtgStoch и любые будущие индикаторы из папки indicators/ctg/ — только для фьючерсов. На акциях и прочих не-фьючерсах атомы контанго (ctgstoch и др.) полностью убираются из Op/Cl: условие не участвует в расчёте (не подменяется обычным Stoch). Spot для контанго подгружается с MOEX ISS только при market=futures. В смешанном портфеле (акции + фьючерсы) одна строка логики применяется по-разному: на GAZP — без CtgStoch, на SiH5 — с CtgStoch (если галочка включена).

Оптимизировать (индикаторы) — перебор непустых комбинаций, остаётся набор с лучшим FINRESP.

9.6. Каталог логик — все встроенные стратегии по умолчанию

В сворачиваемом блоке «Логики» — готовые строки Op/Cl/Regime (те же, что в роботе MultiLogic). У каждой строки: кнопка «i.» (справка), «Сгенерировать сопряжённые логики» (до трёх вариантов с запечёнными @@ReverseSides / @@ReverseSignals: ⇄ стороны, ↔ сигналы, ⇄↔ оба; дубликаты по формуле отбрасываются), «Копировать», «Применить». Параметры @SL, … подставляются из «Параметров управления» при «Рассчитать» и «Применить».

Кнопки каталога: «Восстановить логики по умолчанию» — сброс строк к встроенным из кода; пользовательские ключи U1, U2… исчезают. Выбор в блоке «Логика» сохраняется для id, которые остались во встроенном каталоге (порядок чипов тот же); снимаются только отсутствующие id. Если вы сами очистили выбор (все чипы сняты) — пустой выбор остаётся пустым. Если после сброса не осталось ни одной отмеченной встроенной логики — подставляется UT. Кнопка «Установить параметры по умолчанию» (в «Доп. параметрах») сбрасывает @@/@, объём, комиссию и галочки индикаторов; второй диалог предлагает также сбросить каталог — с той же логикой сохранения выбора.

Импорт/экспорт: «Сохранить каталог» / «Загрузить каталог» — JSON строк логик; «Сохранить протокол» в live — отдельный HTML-журнал (§12). При загрузке протокола расчёта в форму восстанавливаются параметры и стек логик; при сбое пересчёта агрегат FINRESP из файла может остаться в нижнем блоке.

Общие принципы

  • Regime(LinReg;…) — режим рынка по наклону линейной регрессии за @LR свечей. Entry=MatchSide — вход только по стороне тренда; Entry=FlatOnly — только в боковике (мертвая зона наклона SlopeDead). OnFlip=Close — при смене режима закрытие по условию Cl с суффиксом OnFlip(Close).
  • Op(…) — открытие позиции; Cl(…) — закрытие. Long / Short — направление.
  • Ab / Bl у индикатора — цена выше / ниже линии (close относительно SMA, полосы Bollinger, канала LinReg и т.д.).
  • LinReg — канал: центр регрессии ± Dev (в L1…L5) или ± K×ATR (в UT/UCT). Сигналы AbUp/BlLo — пробой верхней/нижней границы; AbLinK/BlLinK — цена относительно канала K×ATR; AbRegK/BlRegK — с дрейфом якоря Drift=RegDrift.
  • SL[@SL] / TP[@TP] — стоп и тейк позиции (доля от цены входа или ×ATR — по значению полей).
  • Strict(@Strict) — строгость совпадения всех условий в скобках (0…1).
  • Несколько логик в списке «Логика» — стек приоритета: на баре сначала проверяется верхняя, затем следующая; выход и SL/TP — у логики, открывшей позицию. SMA-логики с непрерывным объёмом в стек с L1…L5 не смешиваются — оставьте одну SMA или одну L-логику.

RND — случайные сделки (тест)

Назначение: контрольный шум — проверка расчёта FINRESP и инфраструктуры без рыночной логики.

Принцип: на каждом баре независимо с вероятностью 12% сигнал Rand(P=12%) на long и отдельно на short. Фиксированные SL[1%] и TP[5%]. Regime не используется.

TBRU — портфель облигаций (состав БПИФ)

Назначение: имитация вложения в облигации из состава БПИФ «Т-Капитал Облигации» (MOEX:TBRU на porti.ru). Это не строка Op/Cl — отдельный тип bond_tbru. Подробно — §9.8.

Кратко: покупка жадно по текущей доходности (сначала самые доходные ISIN из списка фонда), лимит min(100%, Volume% × Max positions) × депозит, rebalance на каждом баре ТФ; в live — перечитывание состава с porti.ru.

TBC — test counter-bokovik (лонг)

Назначение: контртрендовый лонг в боковике — покупка на перепроданности у нижней границы диапазона.

Regime: LinReg с SlopeDead=0.05%, Entry=FlatOnly — только когда рынок «без тренда».

Вход long: close ниже SMA(100) и Stoch %K ≤ 10 и ATR GrOk (рост волатильности) и ADX(14;Max=25)(WkOk) (флэт) и MACD линия ниже сигнальной.

Выход long: close выше SMA(100) и Stoch %K ≥ 90 и MACD выше сигнальной. SL/TP — из @SL/@TP.

UT — Universal Trend

Назначение: торговля по тренду в обе стороны — long на росте, short на падении, с каналом LinReg ±K×ATR.

Regime: LinReg без мёртвой зоны; смена режима может закрывать через OnFlip(Close) на Cl.

Long: вход — цена выше SMA(100), пробой верхней границы канала LinReg(@LR;K=@K) (AbLinK), ATR(14;Gr=3%;Lb=5)(GrOk), ADX(14;Min=25)(TrOk) (сильный тренд). Выход — цена ниже SMA и регрессионный канал с дрейфом якоря на Open (BlRegK) + OnFlip.

Short: зеркально — вход BlLinK под SMA, выход AbRegK над SMA. Ширина канала — поле @K (×ATR), длина — @LR.

UCT — Universal Counter Trend

Назначение: контртренд — long от нижней границы канала при общем контексте, short от верхней; те же LinReg K×ATR и RegDrift, но сигналы инвертированы относительно UT.

Long: вход под SMA + BlLinK + ADX(14;Max=25)(WkOk) (слабый тренд / флэт); выход над SMA + AbRegK OnFlip.

Short: вход над SMA + AbLinK + ADX(14;Max=25)(WkOk); выход под SMA + BlRegK OnFlip.

L5 — LmaxTrend (лонг + short, максимальный набор фильтров)

Назначение: трендовая стратегия с полным стеком индикаторов — для сравнения «тяжёлого» варианта L1/L3.

Regime: трендовый (Entry=MatchSide).

Long: SMA(100) сверху + LinReg AbUp + Bollinger AbUp + VWAP Ab + ATR «рост волатильности» (GrOk) + ADX TrOk (Min=25) + Stoch ≥ 80 + CCI ≥ 100 + Momentum > 0 + MACD > сигнальная. Выход — зеркальные «медвежьи» условия (BlLo, Bl по полосам, Stoch ≤ 20, CCI ≤ −100, MOM < 0, MACD < sig) + OnFlip.

Short: симметрично для нисходящего тренда (BlLo, нижние полосы, Stoch ≤ 20, CCI ≤ −100 и т.д.).

L1 — лонг, тренд

Regime: тренд, вход по стороне тренда.

Вход long: цена выше SMA(100), LinReg AbUp (±Dev=2), ATR GrOk (рост ≥ 3% за 5 баров), ADX(14;Min=25)(TrOk), CCI ≥ 100, MACD > сигнальная.

Выход: цена ниже SMA, LinReg BlLo, CCI ≤ −100, MACD < сигнальная, OnFlip(Close).

L2 — лонг, боковик

Regime: боковик (SlopeDead, FlatOnly).

Вход long: цена выше SMA, Stoch %K ≤ 10 (откат вверх от перепроданности в диапазоне), ATR GrOk, ADX(14;Max=25)(WkOk) — слабый тренд / флэт, MACD > sig.

Выход: цена ниже SMA, Stoch ≥ 90, MACD < sig, OnFlip.

L3 — short, тренд

Зеркало L1 для short: вход под SMA + LinReg BlLo + ATR GrOk + ADX(14;Min=25)(TrOk) + CCI ≤ −100 + MACD < sig; выход над SMA + AbUp + CCI ≥ 100 + MACD > sig, OnFlip.

L4 — short, боковик

Зеркало L2 для short: Regime боковик; вход short при цене ниже SMA, Stoch ≥ 90, ATR GrOk, ADX(14;Max=25)(WkOk), MACD < sig; выход при цене выше SMA и Stoch ≤ 10, OnFlip.

sma_below — «Ниже SMA», объём |Close−SMA|

Назначение: эталонная SMA-стратегия без дискретных Op/Cl — непрерывный набор/сброс объёма.

Принцип: строка SMA(100;Vol)(Bl) — пока close ниже SMA(100), позиция long с объёмом, пропорциональным расстоянию |Close−SMA| (чем дальше ниже — тем больше лот в рамках настроек объёма). Выход при возврате цены к SMA или по SL/TP. Маркеры на графике — при переходе через ноль и по стопам.

sma_above — «Выше SMA», объём |Close−SMA|

То же, но условие (Ab) — удержание long, пока цена выше SMA; объём растёт с отдалением вверх от средней.

sma_corridor_trend — SMA-эталон, тренд, коридор ATR

Принцип: SMA(100;Spread=@SmaCorridor)(Trend) ADX(14;Min=25)(TrOk) — вокруг SMA строится коридор ±@SmaCorridor×ATR. В режиме Trend набор объёма при движении по направлению выхода за коридор (импульс в сторону тренда относительно SMA), только при сильном ADX. Непрерывная позиция, SL/TP из полей.

sma_corridor_anti — SMA-эталон, анти-тренд, коридор ATR

SMA(100;Spread=@SmaCorridor)(Anti) ADX(14;Max=25)(WkOk) — зеркальная логика: объём на откатах против импульса за границу коридора (контртренд относительно SMA+ATR-полосы) при слабом ADX.

FTS — Фьючерс: CtgStoch + Stoch (лонг), контртренд 20↔80

Идея: покупаем «внизу пилы» и продаём «вверху» на боковике фьючерсов, с подтверждением от контанго (спред фьючерс − базовый актив).

CtgStoch: стохастик (14-3-3) по ряду OHLC контанго: для каждого фьючерса подгружается spot с MOEX ISS, на каждом баре fut − spot, по этому ряду считается %K/%D (атом ctgstoch в строке логики). На акциях этот атом не применяется — см. §9.5.

Вход long: CtgStoch.K ≤ 20 и одновременно Stoch.K ≤ 20 инструмента.

Выход long: CtgStoch.K ≥ 80 и одновременно Stoch.K ≥ 80 инструмента. SL/TP — по полям @SL/@TP.

Раньше использовался TotStoch (стохастик по тотальной цене корзины). С 2026-06-17 — только CtgStoch для семейства FTS/FTT/FTS_S/FTT_S.

FTT — Фьючерс: CtgStoch + Stoch (лонг), тренд 80↔20

Идея: трендовый вариант на той же базе контанго: покупаем, когда спред «разогнан вверх» одновременно по контанго и по инструменту.

Вход long: CtgStoch.K ≥ 80 и одновременно Stoch.K ≥ 80 инструмента.

Выход long: CtgStoch.K ≤ 20 и одновременно Stoch.K ≤ 20 инструмента. SL/TP — по полям @SL/@TP.

FTS_S — Фьючерс: CtgStoch + Stoch (шорт), контртренд 80↔20

Зеркальная версия FTS для short: продаём «вверху пилы», выходим «внизу».

Вход short: CtgStoch.K ≥ 80 и одновременно Stoch.K ≥ 80 инструмента.

Выход short: CtgStoch.K ≤ 20 и одновременно Stoch.K ≤ 20 инструмента. SL/TP — по полям @SL/@TP.

FTT_S — Фьючерс: CtgStoch + Stoch (шорт), тренд 20↔80

Зеркальная версия FTT для short: продаём «внизу» по стохастикам, закрываемся «вверху».

Вход short: CtgStoch.K ≤ 20 и одновременно Stoch.K ≤ 20 инструмента.

Выход short: CtgStoch.K ≥ 80 и одновременно Stoch.K ≥ 80 инструмента. SL/TP — по полям @SL/@TP.

CML — CM long (CMA + LinReg на входе)

Назначение: кастомная средняя CMA с показателем степени @CmaPow и длиной @CmaLen — только лонг.

Вход long: цена выше CMA и LinReg AbUp (пробой верхней границы регрессии, Dev=2). Regime в строке нет — фильтр только по CMA+LinReg на Op.

Выход: цена ниже CMA, OnFlip(Close) — закрытие при пересечении CMA вниз.

CMS — CM short (CMA + LinReg на входе)

Зеркало CML для short: вход — цена ниже CMA + LinReg BlLo; выход — цена выше CMA, OnFlip(Close).

OB_SMA — SMA + стакан (Imb · Spr · Depth)

Назначение: вход по тренду SMA(100) и три условия по снимку стакана; выход по SMA или anti-imb. Профиль mixed — в FINRESP работает только SMA-часть (§9.3.1).

Вход long: SMA(100)(Ab) AND OB.Imb(…;Mode=trend)(BuyOk) AND OB.Spr(…)(Tight) AND OB.Depth(…)(Liquid).

Выход long: SMA(100)(Bl) OR OB.Imb(…;Mode=anti)(SellOk).

OB_ONLY — только стакан · live

Назначение: вход и выход только по OB.* без индикаторов свечей. Профиль only — кнопка «Рассчитать» выдаёт предупреждение и не считает FINRESP; используйте live с T-Bank.

Для UT, UCT, L1…L5 канал LinReg на графике — фиолетовые линии, если LinReg включён в «Индикаторы» и входит в выбранную логику. Текстовые поля Op/Cl/Regime в каталоге редактируются как в роботе; Применить пересчитывает с подстановкой @-параметров.

9.8. TBRU — портфель облигаций (состав фонда)

Логика TBRU в списке «Логика» моделирует покупку отдельных облигаций из состава БПИФ Т-Капитал Облигации (тикер TBRU на MOEX). Состав (ISIN и доли) берётся с porti.ru; в каталоге калькулятора — резервная копия ~67 бумаг с номиналом, купоном и весом.

Что не нужно для TBRU

  • Выбор акций/фьючерсов в §4 — расчёт идёт по одному виртуальному инструменту TBRU.
  • Строки Op/Cl, Regime, индикаторы SMA/Stoch и т.д. — не участвуют (тип bond_tbru).
  • Стакан OB.* и @OBT — не применяются.

Лимит вложений (deploy)

Без маржи, как у остального портфеля (§9.1.1):

deploy ₽ = депозит × min(100%, Volume% × Max positions)

Пример: депозит 10 000 ₽, Volume 10%, Max positions 10 → deploy 10 000 ₽ (100%). При Volume 10% и Max positions 5 → deploy 5 000 ₽.

FINRESP по TBRU = итоговая equity портфеля облигаций (кэш + переоценка + купоны) минус deploy на старте периода.

Reconcile: deploy-конверт и минимум сделок

Учитываются только облигации на счёте (ISIN из фонда или распознанный bond-ISIN; акции и прочее не трогаются). На каждом rebalance:

  1. Deploy-cap = min(100%, Volume% × MaxPos) × депозит.
  2. Список A (счёт): облигации на счёте, сортировка по цене ↑, round-robin +1 шт. пока сумма ≤ deploy — это «конверт». Штуки вне конверта (дорогие / не влезли) — не трогать.
  3. Список B (цель фонда): жадно по доходности (yield), с пустого портфеля, в тот же deploy-cap.
  4. Сличение: цель по ISIN = B + штуки вне конверта; сделка только дельта (3→4 докупить 1, 2→1 продать 1). Совпало — без сделки.
  5. Порядок заявок: сначала продажи, покупки — от более доходных.

«Сирота» (ISIN нет в porti) внутри конверта — продаётся; вне конверта — остаётся на счёте.

Веса porti.ru — при равной доходности; при покупке приоритет — yield.

Расчёт «Рассчитать» (бэктест)

  • На каждом баре выбранного Таймфрейма — rebalance к целевым позициям, начисление купонов (дискретно в «купонные» дни, не ежедневный шум), погашение по номиналу в дату maturity (если задана).
  • Цены облигаций в симуляции — из каталога (чистая ~98% номинала) + линейный НКД; в день купона — откупон (зубчик грязной цены вниз).
  • Результат в блоке FINRESP — одна строка TBRU; в протоколе — агрегат bySec.TBRU, суммы покупок/продаж, число позиций.
  • Портфельный Stopper (@@SL/@@TP) и позиционные @SL/@TP к облигациям TBRU не применяются — только rebalance и купоны.

Графики TBRU (§10.3)

После расчёта: кривая equity портфеля облигаций и отдельные mini-графики по каждой купленной бумаге (ISIN). На графике облигации — ▲ покупка, ▼ продажа, жёлтый вертикальный штрих — день купона; цена — грязная (с НКД).

Live и песочница

  • На каждом опросе по ТФ: обновление состава с porti.ru (до 3 попыток на бар; при локальной разработке — прокси GET /finresp-tbru-holdings на tech-log сервере :4201), пересчёт целей, reconcile с брокером/песочницей.
  • ISIN вне porti внутри deploy-конверта — цель 0 (продать); вне конверта — не трогаем.
  • На счёте T-Bank читаются только позиции с bond-ISIN (SU… / RU…).
  • В песочнице (фейк) — те же цели; купоны начисляются по модели каталога; цены — из sandbox-котировок облигаций.
  • Параметры @@ReverseSides / ReverseSignals, @@PauseOnDrawdown и индикаторы на TBRU не влияют.

10.1. Графики по инструментам (calc-chart)

После «Рассчитать» (или в live-сессии при обновлении графиков) для каждого инструмента строится отдельный блок с тикером и FINRESP по этой бумаге.

Цена — свечи

  • Зелёная свеча — close ≥ open; красная — close < open.

Линии индикаторов (ленивая загрузка)

После «Рассчитать» на графиках сразу рисуются свечи, маркеры сделок и линия FINRESP инструмента — без тяжёлого пересчёта всех индикаторов на десятках бумаг. Ценовые оверлеи подгружаются по кнопке «Показать индикаторы» на конкретном графике (рядом с «Копировать график»). После загрузки подпись меняется на «Индикаторы на графике». В фоне может идти предпрогрев серий (queueChartIndicatorPrewarm), но отрисовка — только по явному клику.

Рисуются только ценовые линии из выбранных логик и галочек «Индикаторы» (не осцилляторы в отдельной панели):

  • SMA, CMA — оранжевая пунктирная; для SMA-коридора — верх/низ коридора
  • LinReg — фиолетовая (центр и полосы ±Dev или ±K×ATR)
  • Bollinger — голубая (средняя и верх/низ)
  • VWAP — зелёная пунктирная
  • FINRESP инструмента — чёрная линия на правой шкале (₽), после первого △/▲. Значение: eq − eq₀. Подпись FINRESP … ₽ в углу — по видимому окну zoom/pan

Stoch, MACD, CCI и прочие осцилляторы на mini-график не выводятся (только в расчёте Op/Cl). Если индикатор выключен или не входит в выбранные логики — линия не рисуется. Под графиком — легенда. На картинке (и в «Копировать график») слева — тикер.

Кнопки над графиком

Слева направо: тикер → «Копировать график» → «Показать индикаторы» → элементы масштаба. «Копировать график» — PNG видимого фрагмента в буфер (HTTPS или localhost).

После завершения «Рассчитать» страница прокручивается к блоку FINRESP (calc-result-hero), чтобы результаты были в центре экрана.

Маркеры входа и выхода

Маркеры ставятся при смене позиции в симуляции (в т.ч. flip long↔short и SL с повторным входом на одном баре). У △/▲ — подпись: первая строка — имя логики (L1, UT, SMA…), вторая — фрагмент из парсера (Op(Long(...)), Cl(Short(...)), SL[…×ATR] и т.п.). При наведении — полная подсказка.

МаркерЗначение
△ остриём вниз над high свечиВход — зелёный long, красный short; подпись: логика + Op(…)
▲ остриём вверх под low свечиВыход — Cl(…), Op flip, SL/TP или Sma → flat

Размер △/▲ зависит от ширины свечи на экране. Подписи скрываются при очень плотном масштабе (узкие свечи); приблизьте участок колёсиком.

У SMA-логик с непрерывным набором/сбросом объёма маркеры — при переходе позиции через ноль и при SL/TP.

FINRESP, стопы и портфель

  • Бумага без сделок (нет △/▲) — линия FINRESP и подпись в углу не показываются; портфельный stopper не пересчитывает её хвост.
  • SL / TP по позиции — тонкая вертикальная (SL красная, TP зелёная); только на барах с △/▲.
  • SL / TP портфельного стоппера — жирнее, другой пунктир; метка события по портфелю на всех графиках.

Live-сессия (режим «Реальная торговля»)

После выбора кошелька «Реальная торговля T-Bank» графики по инструментам показывают окно с момента начала live-сессии (не весь исторический расчёт). FINRESP на mini-графике и в блоке результатов — от нуля сессии (baseline на первом баре сессии). Цветной фон: зелёный — песочница, розовый — реальная торговля. Вертикальные пунктиры — фактические заявки (синяя покупка, оранжевая продажа).

Снятие галочки — все фейк-позиции и фейк-заявки удаляются, портфель снова показывает текущую реальную сумму T-Bank
Бейдж «Логики»Рядом со статусом — краткий список выбранных логик (UT → L5 +2). Клик раскрывает полный порядок приоритета (как в блоке «Логика» калькулятора). Обновляется при смене выбора
Бейдж статуса«остановлена» / «торговля активна» / «внимание» (оранжевый) / «ошибка» (красный, в т.ч. в зелёной песочнице); в песочнице — «· песочница (фейк)». При старте — подготовка: свечи … и FINRESP N/M… (прогресс liveFinrespBootstrap в тех. информации)
Бейдж версии (pageVersion)Справа от статуса — версия страницы (проверка после Ctrl+Shift+R). Актуальная: 2026-06-19-tbru-delta-reconcile-v30
@@PauseOnDrawdown (toolbar)Галочки в панели live — синхронизируются с «Параметрами управления»: портфельная пауза (§5.2) и @@PauseOnDrawdownPerLogic (отключение отдельных логик без остановки сессии)
Чипы логик «отключена»В свёрнутом выборе логик — пометка у логик, временно исключённых из стека по просадке (PerLogic). Состояние сохраняется в config и live-сессии
Монитор стоповФоновый опрос ~8 с: портфельный @@SL/@@TP, @@PauseOnDrawdown, позиционные SL/TP (уведомления). В песочнице портфельный Stopper закрывает фейк-позиции; на реале — только notify (§9.4)
ПлечоТо же значение, что в доп. параметрах
Свечи liveБрокер (если токен) — по умолчанию: свежие бары с API выбранного брокера (T-Bank или Алор). Только брокер — то же, без MOEX. Только MOEX ISS — бесплатно, но часто отстаёт на 10–20 мин. История для индикаторов: при первом «Начать торговлю» без «Рассчитать» подгружается MOEX/кэш, хвост обновляется брокером
Начать / Остановить торговлюДостаточно выбранных инструментов, логики и объёмов; «Рассчитать» не обязателен — нижний блок только для теоретической оценки на истории. При активной торговле: опрос свечей → хвост FINRESP по каждому тикеру → reconcile (сделки) раньше, чем отрисовка десятков мини-графиков; форма остаётся кликабельной (кнопка «Остановить торговлю» всегда доступна). Интервал опроса: 1м ≈ 30 с, 5м ≈ 45 с…
Бейдж при торговле«T-Bank/MOEX бары до … · опрос HH:MM» — начало последнего бара (МСК); «опрос» — время последнего запроса. При нескольких тикерах расчёт по общему концу (самый отстающий)
Подтверждать тренд по стакану (@OBT / OB.*)Галочка в toolbar live-блока — §9.3 и §9.3.1; фильтр перед заявкой по GetOrderBook
Реверс сторон (Long ↔ Short) @@ReverseSidesДублирует реверс в «Параметры управления». Меняет местами сигналы Op/Cl long↔short (полный своп сторон). В live при смене — закрытие всех позиций, затем переключение
Фин. результат (₽)Текущий портфель минус старт сессии (как если закрыть всё по текущим ценам, с учётом комиссий). В песочнице — по фейк-портфелю
Заявка (свёрнутый блок)Свой выпадающий список Инструмент (из бумаг калькулятора), покупка/продажа, рыночная/лимитная, лоты, «Выставить заявку». При лимите и смене инструмента цена подставляется автоматически: сначала close последней свечи выбранного тикера, иначе последняя цена T-Bank (не остаётся цена прошлого инструмента). Двойной клик в стакане — цена из стакана. Выбор сохраняется в config. Независим от инструмента в «Стакане»
Стакан (свёрнутый блок)Отдельный список Инструмент в красной строке над таблицей (справа — глубина, время, спред; тикер в строке — не селектор). Данные T-Bank GetOrderBook, опрос ~4 с при развёрнутом блоке. Двойной клик по цене (подчёркнутая ячейка bid/ask) — развернуть «Заявку» и заполнить: инструмент стакана, лимит, цена клика, сторона, ровно 1 лот; заявка не отправляется автоматически. Тот же снимок используется для OB.* при reconcile (§9.3.1)
История сделок (свёрнутый блок)Журнал исполнений и активных заявок; в заголовке панели — счётчик · N штук. Кнопка «Сохранить протокол» — скачивание автономного HTML (FIFO-пакеты, открытые лоты, раздел «Логики: включения и отключения»). Данные только из RAM журнала — без повторного запроса пароля брокера; предпросмотр в новой вкладке через blob (не URL SPA). Можно нажимать во время активной торговли
Нереализованные позиции (свёрнутый блок)Только открытые (незакрытые) позиции; в заголовке — · N штук. Без песочницы — остатки T-Bank (робот и ручные): тикер, лонг/шорт, лоты, цены, P/L. После «Начать торговлю» — позиции, изменённые с начала сессии. В песочнице — фейк-позиции с пометкой (фейк). ПКМ — закрыть по рынку/лимитом. Опрос ~6 с
Продать всёЗакрыть все позиции (даже при остановленной торговле)
Тип заявки (автоторговля)Рыночная или лимитная для синхронизации позиций роботом
Портфель (все позиции)Стоимость счёта с позициями (GetPortfolio). В песочнице — подпись «Портфель (фейк, симуляция)»: стартовая сумма на момент включения галочки ± фейк-сделки и переоценка открытых фейк-позиций
Комиссии уплачено (реально)Комиссии брокера из операций T-Bank (не модель калькулятора). В песочнице — «Комиссии (фейк, модель %)» по полю Commission %
Заявки в реальном времениТаблица активных и недавних заявок. В песочнице — «Заявки (фейк, симуляция)» с пометкой (фейк)
TBRU (bond_tbru)Если в стеке логик выбрана TBRU: на опросе — refresh состава porti.ru, жадные цели по доходности, reconcile облигаций (покупка/продажа лотов по ISIN). См. §9.8. Ручная заявка в «Заявке» и TBRU-reconcile работают параллельно — ориентируйтесь на блок «Нереализованные позиции»

Не все тикеры MOEX доступны для торговли через API T-Bank (api_trade_available_flag). При «выбрать все акции» калькулятор предпочитает листинг MOEX (TQBR), проверяет статус до заявки и пропускает недоступные бумаги, продолжая торговлю по остальным. Токен должен иметь право на торговлю (не только чтение).

При live без предварительного «Рассчитать» или при расхождении окон свечей возможно предупреждение «FINRESP пустой» / windowSkipped — свечи и опрос продолжаются, торговля не останавливается; сузьте выбор или нажмите «Рассчитать» для согласования с нижним блоком.
